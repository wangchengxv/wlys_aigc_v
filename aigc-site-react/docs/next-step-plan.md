# 前端改版下一步计划

## 目标

在已经完成全局壳层和核心页面改版的基础上，继续把工程推进到更稳定、更统一、更适合交付和验收的状态。

## 第一阶段：收尾与验收

1. 清理当前剩余的 `lint warnings`，优先收口 hooks 依赖问题，避免技术债继续堆积。
2. 按关键路由做一轮人工回归：
   - `/`
   - `/workspace`
   - `/courses`
   - `/courses/:courseId`
   - `/script-projects`
   - `/script-projects/:projectId`
   - `/admin/media-resources`
   - `/settings`
3. 做角色验收，确认 `ADMIN`、`TEACHER`、`STUDENT` 三套导航和入口符合预期。
4. 检查小屏和移动端表现，重点确认：
   - 左侧导航可折叠
   - 顶部工具条不拥挤
   - 抽屉和弹窗在小屏下可正常使用

## 第二阶段：统一次级页面

1. 将项目相关次级页面统一到新的页面壳层和组件体系中，包括：
   - 资产
   - 预览
   - 导出
   - 配音
   - 口型
   - 成片
2. 将资源和平台管理页统一成同一模板：
   - 标题栏
   - 工具条
   - 紧凑列表或卡片
   - 可折叠帮助提示
3. 继续减少解释性文案，将帮助说明隐藏到折叠区、提示入口或悬浮组件中。

## 第三阶段：合并工作台入口

1. 统一 `/workspace`、`/tools/image`、`/tools/video`、`/tools/image-to-video` 的交互壳层。
2. 使用同一套分段切换或 tabs 承载不同模式，减少重复页面。
3. 复用同一套输入、筛选、结果区组件，避免同类功能分散在多个区域。

## 第四阶段：体验打磨

1. 统一按钮层级、卡片密度、状态标签和空状态样式。
2. 统一“创建/编辑”入口为抽屉或弹窗，不再占用首屏。
3. 继续做减噪处理，删除多余文字、重复标题和重复功能入口。

## 第五阶段：性能与交付

1. 处理当前 `vite build` 的大包 warning，做基础拆包和懒加载优化。
2. 补一版截图验收清单，便于快速确认改版成果。
3. 整理最终交付说明，概括：
   - 已完成的核心改版页面
   - 已沉淀的通用组件
   - 后续可继续扩展的页面范围

## 建议执行顺序

建议下一轮按以下优先级推进：

1. 清理 warnings 并完成关键路由回归
2. 继续统一项目次级页
3. 合并 workspace 多入口壳层
4. 做移动端和体验细节收尾
5. 再处理性能优化与最终交付材料

## 验收标准

### 视觉验收

1. 页面首屏不再被长文案占满。
2. 页面标题区只保留标题、状态和主按钮。
3. 卡片、筛选条、标签、空状态都符合统一的轻量化样式。
4. 不再出现旧的重渐变、重玻璃拟态和强展示型区块。

### 交互验收

1. 创建和编辑操作统一通过抽屉或弹窗进行。
2. 相似功能进入同一个区域，不再分散在多个说明型模块中。
3. 项目流程页以内页 tabs 或步骤方式组织，不再依赖额外侧边工作流说明。
4. 帮助说明默认收起，仅保留必要入口。

### 路由验收

1. `/`
2. `/operations-dashboard`
3. `/workspace`
4. `/courses`
5. `/courses/:courseId`
6. `/script-projects`
7. `/script-projects/:projectId`
8. `/admin/media-resources`
9. `/settings`

### 角色验收

1. `ADMIN` 可以看到完整导航和平台管理入口。
2. `TEACHER` 只看到教学管理、项目生产、资源中心等主入口。
3. `STUDENT` 只看到学习和个人创作相关入口，不出现后台管理入口。

### 技术验收

1. `npm run lint` 通过。
2. `npm run build` 通过。
3. 核心页面不存在明显的 TypeScript 报错或运行时报错。
4. 主要组件可复用，不依赖页面内硬编码布局。
