# 剧本工程功能 — 开发文档

## 1. 背景与术语

见 [概要设计](./概要设计.md) 第 2 节。实现时需注意：**optimize 智能体**负责丰富结构化 JSON；**资产抽取**从 JSON 映射到 `ExtractedAsset`，二者阶段不同，勿混淆。

---

## 2. 迭代范围

### MVP（本交付）

- 修订快照列表、恢复、导入到已有项目（含文本解析）。
- 三阶段 optimize API 与六份提示词（system + user）。
- 预览页：原始 / 完善 / 结构化 JSON + **场景 / 角色 / 道具** 结构化视图（组件化）。
- 前端：修订列表、恢复、optimize 按钮与 loading 状态。

### 次轮

- PDF 导入、修订 diff UI、optimize 批量异步任务。

## 4. 接口契约

### 4.1 导入到已有项目

**`POST /api/v1/script-projects/{projectId}/import`**

- Content-Type: `multipart/form-data`
- 字段：`file`（必填）、`replaceName`（可选，字符串）、`autoRefine`（可选，`true`/`false`）

**成功** `data`：`ScriptProjectAggregate`（与详情接口一致）。

**行为**：解析文件为文本 → 更新项目原始剧本 → 可选自动 refine；若 `autoRefine=true` 需配置文本模型。

---

### 4.2 修订列表

**`GET /api/v1/script-projects/{projectId}/revisions`**

**成功** `data`：`ScriptRevision[]`（按 `revisionIndex` 或 `createdAt` 降序，实现以代码为准）。

**单条 `ScriptRevision` JSON 示例**：

```json
{
  "revisionId": "rev-1730000000000-abc12345",
  "revisionIndex": 3,
  "label": "before-optimize-scenes",
  "kind": "OPTIMIZE_SCENE",
  "createdAt": "2026-03-30T12:00:00Z",
  "refinedMarkdownFileId": "file-xxx",
  "refinedJsonFileId": "file-yyy"
}
```

---

### 4.3 恢复修订

**`POST /api/v1/script-projects/{projectId}/revisions/{revisionId}/restore`**

- Body：空

**成功** `data`：`ScriptDocumentPayload`（当前剧本）。

**行为**：对当前内容再拍一条快照 → 将目标修订的 md/json 写回当前工作副本。

---

### 4.4 三阶段优化

| 路径 | 前置条件 |
|------|----------|
| `POST .../optimize/scenes` | 已存在非空 `structuredScript`（先 refine） |
| `POST .../optimize/characters` | 同上 |
| `POST .../optimize/props` | 同上 |

**成功** `data`：`ScriptDocumentPayload`。

---

### 4.5 错误码约定

| HTTP / code | 场景 |
|-------------|------|
| 400 | 参数缺失、未配置文本模型、未先完善剧本 |
| 404 | 项目或 `revisionId` 不存在 |
| 502 | LLM 返回非 JSON 或调用失败 |

业务错误信息通过 `ApiResponse.message` 返回（与现有 `BizException` 一致）。

## 5. 提示词文件清单

| 文件 | 用途 |
|------|------|
| `prompts/script/optimize-scene-system.md` | 场景/场次拍摄向优化 system |

| `prompts/script/optimize-prop-system.md` | 道具创意 system |
| `prompts/script/optimize-prop-user.md` | 道具 user 模板 |

约束：仅输出严格 JSON；输出需可被服务端解析为对象；字段与 [概要设计](./概要设计.md) 第 5.2 节合并策略一致。

## 6. 测试计划

- **集成**：`ScriptProjectWorkflowIntegrationTest` 扩展：创建 → refine → optimize 之一（可 mock provider）。
- **手工验收**：
  - 创建项目 → refine → 依次点击三种 optimize → 预览页可见新字段；
  - 修改剧本保存 → 修订列表增加条目；
  - 恢复某条修订 → 内容与该快照一致。

## 7. 风险与缓解

| 风险 | 缓解 |
|------|------|
| Token 超限 | 提示词压缩输入（仅传结构化 JSON 字符串 + 必要原文片段） |
| JSON 合并冲突 | 按 `id` 合并，缺失 id 则跳过并打日志 |
| 修订文件增多 | 次轮增加保留策略或归档 |
