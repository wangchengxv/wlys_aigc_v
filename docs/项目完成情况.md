## 5. 当前主要风险

1. 文档、规格勾选状态与代码现状存在局部不一致，尤其是视频剪辑与版本治理相关能力，容易造成交付口径误判。
2. 剧本工程、视频生成、配音、口型、剪辑、导出等链路高度依赖外部模型与本地文件存储，现阶段缺少更强的告警、补偿和成本视图。
3. `aigc-server-go` 已进入并行对照阶段，但项目级文档尚不足以证明其能无差异替代 Java 主后端。
4. 已有看板偏总览展示，尚未形成面向运营治理的资源计量、失败治理和容量管理闭环。

## 6. 建议的下一步

1. 先完成视频剪辑工作台的规格对齐与清单验收，消除“代码已存在但任务未勾选”的状态分裂。
2. 以运营看板为基础补齐成本分析、模型消耗统计、失败补偿和告警，优先解决平台可运营性问题。
3. 明确“版本对比与回滚增强”的边界，把已存在的提示词回滚能力与尚未完成的项目级版本治理拆开描述。
4. 按高校场景继续推进教学治理细化能力，如课程成员、批量导入和班级关联课程。
