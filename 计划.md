前端改版全量收尾计划
Summary
本轮按 1-5 全量收口，不拆成多轮。
保持现有公开路由稳定，尤其保留 /workspace、/tools/image、/tools/video、/tools/image-to-video 四个入口。
当前已确认的基线问题：npm run lint 有 10 个 react-hooks/exhaustive-deps warning；npm run build 通过，但主包仍有一个约 739 kB 的大包 warning。
Key Changes
1. Warnings 与验收基线
清掉全部 10 个 hooks warning，不使用 eslint-disable；统一做法是把复用型 loader 改为 useCallback，一次性异步初始化逻辑直接内联到 useEffect，并补齐稳定依赖如 showToast、各类 load* 方法。
单独修 ProviderHubPage 的 memo 依赖问题，把 modelsForConn 变成稳定 memo 输入，避免依赖数组每次变化。
每完成一批 warning 修复就重跑 npm run lint 与 npm run build，保证始终可回归。
人工回归固定覆盖：/、/workspace、/courses、/courses/:courseId、/script-projects、/script-projects/:projectId、/admin/media-resources、/settings。
角色验收使用登录弹窗内置预设账号完成。
ADMIN 验收标准：能看到并进入工作台、课程、项目、媒体资源、模型配置、组织用户、审计日志、设置等完整后台入口。
TEACHER 验收标准：保留工作台、课程、项目、资源工具、设置；不显示组织治理、审计、模型配置、媒体资源这类管理员入口。
STUDENT 验收标准：只保留学习和个人创作相关入口；不出现后台治理和系统配置入口；课程页与项目页 CTA 只保留个人任务相关动作。
响应式验收固定看 1440 / 1024 / 768 / 390 四档，重点检查左侧导航抽屉、顶部栏换行、筛选条折叠、抽屉/弹窗拥挤和溢出。
2. 项目次级页彻底统一
新建一套项目次级页共享壳层，承接 /script-projects/:projectId/preview、assets、video、dubbing、lip-sync、final-composition、export。
统一页面结构为：ScriptProjectWorkflowNav + 紧凑头部/状态带 + 工具条或筛选条 + 主内容区 + HelpHint，不再保留旧式大段说明块。
把当前仍停留在旧工作流壳层的页面迁到新壳层；导出页继续作为样板页，其他项目页对齐它的密度和信息层级。
所有创建/编辑类动作统一改为抽屉或弹窗承载，不再让表单占首屏。
项目链路内帮助说明默认折叠到 HelpHint，首屏只保留状态、关键动作、结果概览。
复查全局导航，删除“同一任务多处重复入口”的全局快捷项，让流程动作尽量回到项目上下文里。
3. 后台页模板统一
将 admin/media-resources、settings 及同类后台页统一成同一模板：标题栏、工具条、紧凑列表或卡片、折叠帮助说明。
以现有 PageToolbar、CompactFilterBar、StatStrip、HelpHint 为主干收口，不再保留风格明显偏旧的 hero/teaching-panel 结构。
统一空状态、状态标签、按钮主次级、卡片密度和列表行间距，避免后台页之间视觉语言割裂。
路由 handle.description 保持轻量，不重新放回首屏长说明；需要说明时统一下沉到帮助提示。
4. Workspace 多入口合并
四个入口继续保留原路径，但都落到同一套 workspace 壳层，不做重定向。
不再依赖 pathname.includes(...) 推断模式，改成显式路由元数据驱动默认模式，例如给 RouteHandle 增加 workspaceMode 一类字段。
用一个分段切换或 tab 区承载图像、视频、图生视频等模式；共用同一套 PromptPanel、ResultPanel、筛选区和结果操作区。
相同动作只保留一处：生成、重试、收藏、下载、查看历史等共用逻辑放回统一区域，模式差异只体现在当前 tab 的字段和结果卡片。
/workspace 作为综合入口显示完整模式切换；三个 /tools/* 路径只负责落在对应默认 mode，但视觉和交互结构完全一致。
5. 体验打磨与性能交付
统一按钮层级、状态标签、空状态、卡片边距、工具条间距和首屏标题区密度。
补强小屏 CSS，重点覆盖 app-shell、page-toolbar、compact-filter-bar、fixed-panel-dock、ActionDrawer 以及登录/确认/服务商类弹窗。
对重量级路由做懒加载，优先拆项目流程页、后台配置页、画布/图形相关页面、workspace 相关页面。
如果路由懒加载后主包仍超阈值，再在 vite.config.ts 加手工分包，按 vendor/router、graph-canvas、project-workflow、admin-config 这几组切块。
在 aigc-site-react/docs/ 新增 revamp-acceptance-checklist.md，记录路由、角色、视口、截图占位与通过情况。
在 aigc-site-react/docs/ 新增 revamp-delivery-notes.md，总结核心改版页面、可复用组件、已消除的重复入口和后续可扩展范围。
Public Interfaces / Route Contracts
保持现有页面 URL 不变，尤其是 /workspace 和三个 /tools/* 别名。
本轮不改后端接口或数据 schema。
主要前端接口变化是路由元数据扩展，用显式模式字段替代当前的路径字符串推断。
新增的共享壳层和模板组件只作为内部复用层，不引入新的对外 API。
Test Plan
必跑：npm run lint、npm run build。
必回归：/、/workspace、/courses、/courses/:courseId、/script-projects、/script-projects/:projectId、/admin/media-resources、/settings。
补充回归：直接访问管理员专属页，确认 TEACHER / STUDENT 的隐藏与拦截行为一致。
角色回归：分别登录 admin、teacher、student，检查导航、页面入口、空状态、首屏 CTA。
响应式回归：桌面与小屏各拍一轮截图，覆盖导航展开/收起、顶部工具条、抽屉、弹窗、筛选条、固定 dock。
性能回归：确认 build 大包 warning 消失，或至少被拆分为多块且主包明显下降。

角色收口遵循“学生只看学习与个人创作、教师只看教学与项目、管理员看完整后台”的方向。
