# AGENTS - AI 协作规矩（工作手册）

## 1. 目标
本文件定义 Miioo 项目中人类与 AI 代理的统一工作规矩，确保所有实现与《Miioo动画视频创作平台-详细开发文档.md》一致，避免偏航、返工和质量失控。

## 2. 适用范围
- 需求拆解、架构设计、前后端开发、测试设计、文档维护。
- 适用于单个 AI 代理与多代理协作。

## 3. 角色约定
- `Owner`：人类负责人，做最终业务决策与验收。
- `Planner Agent`：把需求转成可执行任务、验收标准、风险清单。
- `Builder Agent`：按任务实现代码或文档。
- `Reviewer Agent`：做红线检查、回归影响分析与测试评审。
- `QA Agent`：维护测试用例和质量门禁执行结果。

## 4. 强制执行流程（每次任务都要走）
1. 先读源文档：`Miioo动画视频创作平台-详细开发文档.md`。
2. 确认任务归属模块：`project` / `script` / `subject` / `storyboard` / `render` / `asset` / `aitask` / `model` / `auth`。
3. 在 `SPECS/` 找对应规格；若缺失先补规格，再改代码。
4. 对照 `RULES.md` 做红线预检。
5. 实现后按 `TESTS.md` 执行最小测试集并记录结果。
6. 更新 `PROGRESS.md`：记录完成项、风险、下一步。

## 5. 任务拆解规则
- 每个任务必须包含：`背景`、`范围`、`输入`、`输出`、`验收标准`、`风险`。
- 任务粒度建议 0.5-1.5 天，避免过大任务不可验证。
- 涉及接口变更时，必须同步更新 `SPECS/api-contracts.md`。
- 涉及数据结构变更时，必须同步更新 `DOCS/data-model.md`。

## 6. 交付定义（Definition of Done）
- 需求：与 `SPECS` 对应条目一致。
- 实现：不违反 `RULES.md` 强制红线。
- 测试：通过 `TESTS.md` 中该模块的最小用例。
- 文档：至少更新 `PROGRESS.md`，必要时更新 `ARCHITECTURE.md` 与 `DOCS/`。

## 7. 沟通与输出格式
- 所有变更说明采用固定结构：
  - `做了什么`
  - `为什么这么做`
  - `影响范围`
  - `验证结果`
  - `待确认问题`
- 若信息不足，不得臆测实现；先提问补齐约束。

## 8. 变更优先级
- P0：安全与越权、数据一致性、任务状态错误。
- P1：核心链路中断（项目、剧本、主体、分镜、导出）。
- P2：功能体验缺陷与性能优化。
- P3：文档与可维护性改进。

## 9. 禁止事项
- 不得绕过项目归属校验（`user_id` / `project_id`）。
- 不得在前端暴露模型 API Key。
- 不得跳过异步任务机制直接阻塞式调用长耗时 AI/渲染能力。
- 不得只改代码不更新对应规格和进度文档。
