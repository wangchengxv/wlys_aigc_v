# 项目工作区后置流程隐藏实施计划

## Summary
- 目标：在 `script-projects` 项目工作区中，先隐藏“视频”之后的后置流程功能（配音、口型、剪辑、导出）。
- 成功标准：
  - 工作区左侧流程导航不再显示以上 4 个步骤。
  - 用户即使手动输入后置流程 URL，也会被拦截并跳转到视频页。
  - 保留可恢复能力：通过集中开关控制是否启用后置流程，不做永久删除。
- 范围：仅 `script-projects` 工作区流程页；不改项目列表/项目详情中的其它入口文案与卡片。

## Current State Analysis
- 流程步骤定义位于 `aigc-site-react/src/components/script/scriptProjectWorkflow.ts`，当前顺序为：全局设定 → 总览 → 剧本 → 资产 → 视频 → 配音 → 口型 → 剪辑 → 导出。
- 工作区侧边栏与上下步按钮依赖该步骤列表：
  - `aigc-site-react/src/components/script/ScriptProjectWorkflowNav.tsx`
  - `aigc-site-react/src/components/script/ProjectSubpageShell.tsx`
- 相关路由在 `aigc-site-react/src/router.tsx` 明确注册，后置流程对应：
  - `/script-projects/:projectId/dubbing`
  - `/script-projects/:projectId/lip-sync`
  - `/script-projects/:projectId/final-composition`
  - `/script-projects/:projectId/export`
- 现状问题：即使前端导航隐藏，用户仍可通过直链访问后置页面，因此需要“导航隐藏 + 路由拦截”双层控制。

## Proposed Changes

### 1) 新增集中开关（可恢复）
- 文件：`aigc-site-react/src/config/scriptProjectFeatureFlags.ts`（新增）
- 变更内容：
  - 新增 `SCRIPT_PROJECT_FEATURE_FLAGS` 常量。
  - 提供布尔开关 `enablePostVideoWorkflow`（初始值设为 `false`，表示当前隐藏）。
  - 提供受控步骤 key 集合，至少覆盖：`dubbing`、`lip-sync`、`edit`、`export`。
- 目的：
  - 把“先隐藏”做成可配置策略，后续恢复仅改一处。

### 2) 流程步骤按开关动态裁剪
- 文件：`aigc-site-react/src/components/script/scriptProjectWorkflow.ts`
- 变更内容：
  - 保留完整的“全量步骤定义”。
  - 导出当前生效的 `scriptProjectWorkflowSteps`：当 `enablePostVideoWorkflow=false` 时，过滤掉后置 4 步。
  - `getScriptProjectWorkflowCurrentIndex` 继续基于“生效步骤”计算，保证导航进度和上下步逻辑自动收敛。
- 目的：
  - 同步影响侧边栏与 `ProjectSubpageShell` 上下步，无需在多个组件硬编码隐藏条件。

### 3) 路由层增加后置流程拦截（防直链访问）
- 文件：`aigc-site-react/src/router.tsx`
- 变更内容：
  - 新增工作区路由守卫组件（如 `ScriptProjectWorkflowFeatureGate`）：
    - 读取 `projectId` 参数；
    - 当后置流程关闭时，访问后置 4 条路由统一 `<Navigate replace>` 到 `/script-projects/:projectId/video`；
    - 当开关打开时，正常渲染原页面。
  - 将 4 个后置流程 route 的 `element/lazy` 包装到该守卫中（不改其页面实现文件）。
- 目的：
  - 保障“UI隐藏 + 路由禁用”一致性，满足手动输入 URL 的禁用诉求。

### 4) 增加最小回归测试（聚焦开关与路由判定逻辑）
- 文件建议：
  - `aigc-site-react/src/components/script/scriptProjectWorkflow.test.mjs`（新增）
  - 或新增纯函数文件 `aigc-site-react/src/lib/scriptProject/workflowFeatureGate.ts` + 对应 `*.test.mjs`
- 变更内容：
  - 断言开关关闭时：步骤中不包含 `dubbing/lip-sync/edit/export`。
  - 断言开关开启时：步骤恢复完整。
  - 路由守卫相关逻辑至少覆盖“后置路径命中→跳转视频页”的判定。
- 目的：
  - 避免后续改动把隐藏策略意外恢复或破坏。

## Assumptions & Decisions
- 已确认决策：
  - 隐藏层级：`UI + 路由都禁用`。
  - 实现方式：`加开关便于恢复`。
  - 范围：`仅 script-projects 工作区流程`。
- 实施假设：
  - 不删除后置页面源码与接口调用，仅做入口与访问门控。
  - 关闭后置流程时，默认统一回落到“视频”步骤作为可用终点。

## Verification Steps
1. 本地启动前端后，进入任意项目工作区，确认左侧流程仅到“视频”为止，不再出现“配音/口型/剪辑/导出”。
2. 在工作区页底部“上一步/下一步”连续跳转，确认不会进入后置步骤。
3. 手动访问以下 URL，均应跳转到 `/script-projects/:projectId/video`：
   - `/script-projects/:projectId/dubbing`
   - `/script-projects/:projectId/lip-sync`
   - `/script-projects/:projectId/final-composition`
   - `/script-projects/:projectId/export`
4. 将开关改为开启（`enablePostVideoWorkflow=true`）后，确认步骤和路由行为完全恢复。
5. 运行新增/受影响测试，确认通过；执行一次类型检查或 lint，确认无新增报错。
