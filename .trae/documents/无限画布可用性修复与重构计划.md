# 无限画布可用性修复与重构计划

## 一、Summary
- 目标：修复“无限画布不可正常使用”的交互与数据稳定性问题，并彻底重做页面信息架构，改为“画布主视图 + 可折叠抽屉”。
- 范围：前后端一起改，优先保障拖拽/缩放、节点操作、保存同步、Comfy 回显四条链路可用。
- 结果形态：默认专注画布编辑，工具与状态收纳至抽屉；执行结果默认折叠，提交后自动展开并高亮状态。

## 二、Current State Analysis

### 1) 前端可用性风险（核心）
- `aigc-site-react/src/components/canvas/ComfyLikeCanvas.tsx`
  - 当前 `graph.onAfterChange`、`pointerup`、`wheel`、快捷键等多入口都会触发 `saveSnapshot -> setState -> saveCanvasRemote`，保存频率过高，易造成交互期卡顿和状态抖动。
  - 首次恢复逻辑优先取本地缓存，否则 `syncFromRemote()` 只拉取“最近一条”，未显式绑定当前项目，跨项目时可能恢复到错误草稿。
  - 工具区信息密度过高（按钮、状态卡、模板卡、节点卡、提示文案全部同屏），与“画布编辑”主任务竞争注意力。

### 2) 前端样式冲突风险（界面凌乱）
- `aigc-site-react/src/styles/global.css` 与 `aigc-site-react/src/styles/components.css`
  - `comfy-canvas` 系列选择器在两处重复定义（例如 `.comfy-canvas__toolbar`、`.comfy-canvas__viewport`），且 `components.css` 在 `main.tsx` 中后加载，导致样式覆盖不可控。
  - 页面已经从旧 `canvas-page` 迁移到 `canvas-tool-page`，但 `components.css` 仍保留大量旧画布样式与扩展规则，增加维护复杂度和视觉不一致。

### 3) 后端接口语义不足（同步稳定性）
- `aigc-server/src/main/java/com/example/aigc/controller/CanvasGraphController.java`
- `aigc-server/src/main/java/com/example/aigc/service/CanvasGraphService.java`
- `aigc-server/src/main/java/com/example/aigc/repository/jpa/SpringDataRepositories.java`
  - 列表接口仅按 owner 返回最新记录，缺少按 `projectId` 或草稿标识过滤的显式语义，前端难以“准确恢复当前工作上下文”。
  - 当前创建/覆盖策略以可选 `id` 为主，缺少面向“项目草稿唯一键”的便捷读取入口。

## 三、Proposed Changes

### A. 前端重构（画布主视图 + 抽屉）
- 文件：`aigc-site-react/src/components/canvas/ComfyLikeCanvas.tsx`
- 改动：
  - 重排布局为三层：`Canvas Stage`（主画布）/ `Top Compact Bar`（极简关键操作）/ `Drawer`（模板、节点、保存状态、执行结果）。
  - 新增抽屉状态机：`closed | tools | status | result`，默认 `closed`；点击工具按钮展开对应抽屉；执行提交后自动切到 `result`。
  - 精简常驻操作为：保存、提交、适配视图、抽屉开关；导入导出/连通性检查/清缓存进入“更多操作”分组（抽屉内）。
  - Comfy 结果区改为默认折叠（抽屉内容），执行结束后自动展开并显示成功/失败摘要、图片网格。
- 原因：
  - 将“编辑画布”作为唯一主焦点，降低信息噪音。
  - 符合用户偏好：彻底重做 + 可折叠抽屉 + 结果默认折叠。

### B. 前端状态与保存链路修复（交互优先）
- 文件：`aigc-site-react/src/stores/canvasGraphStore.ts`
- 改动：
  - 引入“本地即时、远端节流”双轨保存：本地 `localStorage` 立即落盘，远端保存改为防抖/节流（如 600-1000ms）与单飞队列（in-flight + latest pending）。
  - 标准化远端状态：`idle | saving | saved | error` 保留，但加上“最近成功时间”和“重试入口”支持。
  - `clear()` 只清除当前草稿上下文（含 remoteId 映射），避免误清其他上下文数据。
- 文件：`aigc-site-react/src/components/canvas/ComfyLikeCanvas.tsx`
- 改动：
  - 缩减 `saveSnapshot()` 触发源：高频拖拽过程只标记 dirty，不直接发远端请求；在交互结束（pointerup / idle）集中提交。
  - 统一快捷键与提示文案（确保文案与实际行为一致）。
- 原因：
  - 解决“拖拽缩放异常、节点操作异常”常见根因（高频保存导致主线程与网络争用）。

### C. 前端远端恢复语义修复（避免串草稿）
- 文件：`aigc-site-react/src/lib/graph/canvasRemoteApi.ts`
- 改动：
  - 扩展列表查询参数（至少支持 `projectId` 过滤；可选 `id` 精确读取）。
  - 统一请求方法，保留兼容默认行为（未传参数时取 owner 最新）。
- 文件：`aigc-site-react/src/stores/canvasGraphStore.ts`
- 改动：
  - `syncFromRemote` 支持按当前 `projectId`/显式草稿上下文加载，不再盲取全局最新一条。
- 文件：`aigc-site-react/src/pages/CanvasPage.tsx`
- 改动：
  - 传递当前项目上下文到画布恢复流程，保障项目切换时读取正确草稿。
- 原因：
  - 解决“保存同步异常、刷新恢复错画布”。

### D. 后端接口增强（与前端上下文对齐）
- 文件：`aigc-server/src/main/java/com/example/aigc/controller/CanvasGraphController.java`
- 改动：
  - 列表接口增加可选 `projectId` 参数（必要时可追加 `id`）。
- 文件：`aigc-server/src/main/java/com/example/aigc/service/CanvasGraphService.java`
- 改动：
  - 增加按 owner + projectId 的过滤读取逻辑（按 `updatedAt` 倒序后分页）。
  - 保持旧调用兼容：未传过滤条件时行为不变。
- 文件：`aigc-server/src/main/java/com/example/aigc/repository/CanvasGraphRepository.java`
- 文件：`aigc-server/src/main/java/com/example/aigc/repository/jpa/JpaCanvasGraphRepository.java`
- 文件：`aigc-server/src/main/java/com/example/aigc/repository/jpa/SpringDataRepositories.java`
- 改动：
  - 补充对应 JPA 查询方法（owner + projectId + updatedAt desc）。
- 原因：
  - 前端需要稳定可预期的数据语义，避免“最近一条”误命中。

### E. 样式体系收敛（彻底清理凌乱来源）
- 文件：`aigc-site-react/src/styles/global.css`
- 文件：`aigc-site-react/src/styles/components.css`
- 改动：
  - 保留一处 `comfy-canvas` 样式源（建议 `global.css`），删除另一处重复/过时规则。
  - 删除旧 `canvas-page` 残留样式块（仅保留仍被其他页面引用的选择器）。
  - 为抽屉式布局新增明确命名规则（如 `comfy-canvas__drawer*`），避免与旧类名相互覆盖。
- 原因：
  - 消除重复定义导致的视觉抖动、层级混乱和后续维护成本。

### F. 路由文案与页面语义一致性
- 文件：`aigc-site-react/src/router.tsx`
- 改动：
  - 更新 `/canvas` 的 handle 描述，从“预留空间”改为“正式可用的节点编排画布工具”，与实际功能一致。
- 原因：
  - 避免产品文案与实际状态不一致，减少用户认知偏差。

## 四、Assumptions & Decisions
- 已确认决策：
  - 改造力度：彻底重做页面。
  - 布局策略：可折叠抽屉为主，画布优先。
  - 范围：前后端一起改。
  - 结果区：默认折叠，提交后自动展开。
- 实施假设：
  - 现有 `@comfyorg/litegraph` 版本维持不变（`^0.17.2`），优先在现有依赖上修复。
  - 后端数据库模型不新增表字段，本次通过查询语义和接口参数增强解决上下文恢复问题。

## 五、Verification Steps

### 1) 前端交互回归（手动）
- 进入 `/canvas`，验证：
  - 默认聚焦画布；工具/状态/结果在抽屉中。
  - 拖拽、滚轮缩放、快捷键（适配视图/删除）流畅，持续操作无明显卡顿。
  - 添加节点、连线、删除、模板切换可正常保存与恢复。

### 2) 保存与恢复回归（手动）
- 场景 A：无项目绑定草稿，刷新后恢复正确。
- 场景 B：绑定项目 A 保存，再切项目 B 保存，反复切换后仍各自恢复正确草稿。
- 场景 C：模拟远端失败（断网/接口报错），本地缓存不丢失，状态提示可重试。

### 3) Comfy 提交流程（手动）
- 提交后：
  - 抽屉自动切到结果视图。
  - 队列中、成功、失败三种状态文案准确。
  - 有图片时正确展示网格预览，无图片时展示清晰占位说明。

### 4) 后端接口验证
- 验证 `/api/v1/canvas` 在有/无 `projectId` 参数时返回符合预期且兼容旧行为。
- 验证 owner 维度权限仍有效，无法读取其他用户画布。

### 5) 静态检查
- 前端执行 `npm run lint`（在 `aigc-site-react`）。
- 后端执行项目既有编译/测试流程（最少保证 canvas 相关改动可编译通过）。

## 六、交付顺序（执行阶段）
1. 后端接口与仓储过滤能力增强（先保证数据语义稳定）。
2. 前端 store 保存/恢复链路修复（先解“不可用”根因）。
3. `ComfyLikeCanvas` 结构重做为抽屉式布局。
4. 样式去重与旧规则清理。
5. 路由文案、回归验证与收尾。
