## Summary

将「上传图片/参考图」窗口改为仅在用户进入「图生视频」入口（/tools/image-to-video，对应 `workspaceVariant === 'image-to-video'`）时展示与校验；其他入口（综合创作的视频模式、文生视频工具页）不展示该上传窗口，也不要求用户上传图片。

## Current State Analysis

- 参考图上传窗口由 [PromptPanel.tsx](file:///Users/xingyi/Downloads/AIGC_%E5%B0%8F%E4%BA%91%E9%9B%80/AIGC_university_%E5%89%AF%E6%9C%AC/aigc-site-react/src/components/workspace/PromptPanel.tsx#L608-L625) 在 `mode === 'video'` 时无条件渲染 [VideoReferenceImageField.tsx](file:///Users/xingyi/Downloads/AIGC_%E5%B0%8F%E4%BA%91%E9%9B%80/AIGC_university_%E5%89%AF%E6%9C%AC/aigc-site-react/src/components/workspace/VideoReferenceImageField.tsx)。
- 提交前校验在 [advancedMedia.ts](file:///Users/xingyi/Downloads/AIGC_%E5%B0%8F%E4%BA%91%E9%9B%80/AIGC_university_%E5%89%AF%E6%9C%AC/aigc-site-react/src/lib/workspace/advancedMedia.ts#L287-L343)：
  - 只要 `needVid === true` 且 `workspaceVideoNeedsFirstFrameImage(finalVideoModel) === true`，就会强制要求 `videoReferenceImageUrl` 合法。
- 路由层已经区分「文生视频」与「图生视频」入口：见 [WorkspacePage.tsx](file:///Users/xingyi/Downloads/AIGC_%E5%B0%8F%E4%BA%91%E9%9B%80/AIGC_university_%E5%89%AF%E6%9C%AC/aigc-site-react/src/pages/WorkspacePage.tsx#L20-L25)；并通过 `workspaceVariant` 注入到 `PromptPanel`。

因此当前问题是：在非图生视频入口也会出现参考图上传 UI，并且某些模型名会触发“必须上传参考图”的校验，导致文生视频/综合创作的视频模式被不必要地要求上传图片。

## Proposed Changes

### 1) 仅在图生视频入口展示参考图上传窗口

修改 [PromptPanel.tsx](file:///Users/xingyi/Downloads/AIGC_%E5%B0%8F%E4%BA%91%E9%9B%80/AIGC_university_%E5%89%AF%E6%9C%AC/aigc-site-react/src/components/workspace/PromptPanel.tsx)：

- 新增布尔值 `isImageToVideoEntry = workspaceVariant === 'image-to-video'`。
- 将 `VideoReferenceImageField` 的渲染条件从 `mode === 'video'` 改为 `mode === 'video' && isImageToVideoEntry`。
- 将视频区块的提示文案（当前为“可随时切换：文生视频/图生视频 …”）改为按入口展示：
  - `isImageToVideoEntry`：保留图生视频 API 文档提示；
  - 非图生视频入口：提示“如需图生视频请切换到「图生视频」入口”，避免误导。
- 为避免用户从图生视频入口切走后状态残留，添加一个 `useEffect`：当 `!isImageToVideoEntry` 时，自动 `setVideoReferenceImageUrl('')`。

### 2) 非图生视频入口不触发“必须参考图”的图生视频校验

修改 [advancedMedia.ts](file:///Users/xingyi/Downloads/AIGC_%E5%B0%8F%E4%BA%91%E9%9B%80/AIGC_university_%E5%89%AF%E6%9C%AC/aigc-site-react/src/lib/workspace/advancedMedia.ts)：

- 扩展 `WorkspaceAdvancedValidationParams`，新增参数：
  - `allowVideoFirstFrameImage?: boolean`
- 调整 `getWorkspaceAdvancedValidationError` 的视频分支逻辑：
  - 当 `needVid && workspaceVideoNeedsFirstFrameImage(finalVideoModel)`：
    - 若 `allowVideoFirstFrameImage !== true`：直接返回引导错误（例如“该视频模型为图生视频模型，请切换到「图生视频」入口”），避免在文生视频/综合入口要求上传图片；
    - 若允许：沿用现有逻辑，继续要求 `videoReferenceImageUrl` 合法。

对应地，在 [PromptPanel.tsx](file:///Users/xingyi/Downloads/AIGC_%E5%B0%8F%E4%BA%91%E9%9B%80/AIGC_university_%E5%89%AF%E6%9C%AB/aigc-site-react/src/components/workspace/PromptPanel.tsx#L236-L247) 调用校验时传入：

- `allowVideoFirstFrameImage: isImageToVideoEntry`

### 3) 限制非图生视频入口的可选视频模型（避免选到必须参考图的模型）

修改 [PromptPanel.tsx](file:///Users/xingyi/Downloads/AIGC_%E5%B0%8F%E4%BA%91%E9%9B%80/AIGC_university_%E5%89%AF%E6%9C%AC/aigc-site-react/src/components/workspace/PromptPanel.tsx#L149-L190) 的视频模型列表加载逻辑：

- 当 `!isImageToVideoEntry` 时，对 `getVideoModels()` 返回的 `options` 进行过滤：移除 `workspaceVideoNeedsFirstFrameImage(modelId) === true` 的模型。
- 若过滤导致当前选中 `videoModel` 不在新列表中，则自动回退到：
  - 服务端默认模型（若仍存在于过滤后的列表）；
  - 否则回退到过滤后的第一个模型；
  - 若列表为空则使用现有默认兜底（保持与当前代码一致的容错策略）。
- 对 `videoModelInputMode === 'custom'` 的情况：由第 2) 的校验逻辑兜底，阻止用户手动输入图生视频模型ID后在文生视频入口提交。

### 4) 更新单测

修改 [advancedMedia.test.mjs](file:///Users/xingyi/Downloads/AIGC_%E5%B0%8F%E4%BA%91%E9%9B%80/AIGC_university_%E5%89%AF%E6%9C%AC/aigc-site-react/src/lib/workspace/advancedMedia.test.mjs)：

- 新增用例：`allowVideoFirstFrameImage: false` 且 `finalVideoModel` 命中 `workspaceVideoNeedsFirstFrameImage` 时，应返回“请切换到图生视频入口”的错误。
- 新增/补充用例：`allowVideoFirstFrameImage: true` 时，仍会在缺少参考图时返回现有“图生视频需填写参考图…”错误。

## Assumptions & Decisions

- “用户选择图生视频功能”在本项目中等价于进入路由 Tab「图生视频」（`workspaceVariant === 'image-to-video'`），而非仅仅在同一页面里切换某个开关。
- `workspaceVideoNeedsFirstFrameImage()` 命中的模型集合视为“图生视频（需首帧/参考图）模型”；因此在非图生视频入口应当隐藏并阻止提交，以匹配“其他界面不要求必须上传图片”的产品预期。

## Verification

- 单元测试：在 `aigc-site-react` 目录执行 `npm run test:workspace-advanced-media`。
- 本地手测（启动 `npm run dev` 后）：
  - `/tools/image-to-video`：展示参考图上传窗口；选择 Vidu/Moark/Wan-i2v 等模型时，不上传参考图会被拦截。
  - `/tools/video` 与 `/workspace` 的视频模式：不展示参考图上传窗口；视频模型下拉不出现 Vidu/Moark/Wan-i2v 等“需参考图”模型；若手动输入此类模型ID，提交会提示切换到图生视频入口。
