# 视频生成返回结果缺失问题分析与修复计划

## 问题描述
前端调用视频生成接口返回成功状态，但 `videoResults` 中没有返回可访问的视频 URL，只有内部文件路径。

### 当前返回数据分析
```json
{
  "status": "SUCCESS",
  "videoResults": ["/api/v1/files/_aigc_workspace__7d905b042069441d8bbbc96fd01f1a03"],
  "latencyMs": 0,  // ⚠️ 可疑：执行时间未被记录
  "persistedVideoFileIds": ["_aigc_workspace__7d905b042069441d8bbbc96fd01f1a03"]
}
```

### 问题特征
1. `videoResults` 返回的是内部文件路径而非可访问的公开 URL
2. `persistedVideoFileIds` 返回的是不带前缀的文件 ID
3. 两者不匹配：应该 `videoResults` 包含完整路径，`persistedVideoFileIds` 只包含 ID

## 根因分析

### 核心问题：persistWorkspaceGenerationFiles 方法中的逻辑缺陷

查看 `GenerationServiceImpl.java` 第 2716-2751 行的代码逻辑：

```java
if (videoResults != null) {
    for (int i = 0; i < videoResults.size(); i++) {
        String url = videoResults.get(i);
        
        // ⚠️ BUG 在这里：如果不是 http(s) URL，直接使用原值
        if (!trimmed.startsWith("http://") && !trimmed.startsWith("https://")) {
            displayVid.add(trimmed);  // 直接使用内部路径
            continue;
        }
        
        try {
            // 下载并存储...
            displayVid.add(localAssetFileService.toPublicUrl(rec.fileId));
        } catch (Exception ex) {
            displayVid.add(trimmed);  // 失败时也使用原值
        }
    }
}
```

**问题**：
1. 如果原始 URL 不是 http(s) 格式（例如已经是 `/api/v1/files/...` 相对路径），代码直接跳过下载步骤
2. 直接使用原始值而不是调用 `toPublicUrl()` 生成正确的公开 URL
3. 导致前端收到的 `videoResults` 是内部路径而不是可访问的 URL

## 修复方案

### 方案：始终生成可公开访问的 URL

修改 `persistWorkspaceGenerationFiles` 方法的逻辑：
- 对于非 http(s) URL 的情况，也应该调用 `toPublicUrl()` 生成正确的公开 URL
- 如果原始 URL 已经是完整公开路径，应该直接使用
- 如果原始 URL 是文件 ID 或相对路径，需要转换

```java
String trimmed = url.trim();
// 检查是否是公开 URL（http(s) 开头或是 /api/v1/files/ 格式）
boolean isPublicUrl = trimmed.startsWith("http://") 
                    || trimmed.startsWith("https://") 
                    || trimmed.startsWith("/api/v1/files/");

if (!isPublicUrl) {
    // 非公开 URL，需要下载并存储
    try {
        StoredFileRecord rec = localAssetFileService.storeRemote(
                WorkspaceConstants.WORKSPACE_PROJECT_ID,
                "workspace-gen/" + taskId + "/vid-" + i + ".mp4",
                "video/mp4",
                trimmed
        );
        scriptProjectService.upsertFile(aggregate, rec);
        persistedVid.add(rec.fileId);
        displayVid.add(localAssetFileService.toPublicUrl(rec.fileId));
    } catch (Exception ex) {
        // 如果下载失败，尝试将其作为文件 ID 处理
        displayVid.add(localAssetFileService.toPublicUrl(trimmed));
    }
} else {
    // 已经是公开 URL，直接添加到结果
    displayVid.add(trimmed);
}
```
