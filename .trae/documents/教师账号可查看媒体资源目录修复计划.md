# 教师账号可查看媒体资源目录修复计划

## Summary
- 目标：修复“教师账号访问媒体资源功能时报‘只有管理员可以查看媒体资源目录’”的问题。
- 成功标准：教师登录后访问 `/admin/media-resources` 可正常返回并展示媒体资源数据；管理员能力不受影响；学生仍不可访问。

## Current State Analysis
- 前端导航与页面守卫已允许教师访问媒体资源：
  - `aigc-site-react/src/components/layout/TopNav.tsx` 中 `媒体资源中心` 角色为 `ADMIN/TEACHER`。
  - `aigc-site-react/src/components/layout/AppShellNav.tsx` 中 `媒体资源` 角色为 `ADMIN/TEACHER`。
  - `aigc-site-react/src/pages/MediaResourcesPage.tsx` 页面守卫为 `ADMIN/TEACHER`。
- 真实拦截发生在后端服务层：
  - `aigc-server/src/main/java/com/example/aigc/service/MediaResourceService.java` 的 `listRecent()` 当前调用 `authorizationService.requireAdmin(actor, "只有管理员可以查看媒体资源目录")`，会导致教师 403。
- 权限工具能力已具备可复用方法：
  - `aigc-server/src/main/java/com/example/aigc/service/AuthorizationService.java` 提供 `requireTeachingManager()`，语义为“教师或管理员可访问”。

## Proposed Changes
- 修改后端媒体资源目录访问校验（核心修复）
  - 文件：`aigc-server/src/main/java/com/example/aigc/service/MediaResourceService.java`
  - What：将 `requireAdmin(...)` 替换为 `requireTeachingManager(...)`，并同步错误文案为“只有教师或管理员可以查看媒体资源目录”。
  - Why：与当前产品权限策略对齐（教师应可访问媒体资源），消除前后端权限不一致。
  - How：仅调整 `listRecent()` 中一处授权调用，不改动查询逻辑与返回结构。
- 补充/调整集成测试，锁定权限边界
  - 文件：`aigc-server/src/test/java/com/example/aigc/AuthIntegrationTest.java`（已有媒体资源接口用例）
  - What：新增或补充分角色断言：
    - 教师访问 `/api/v1/admin/media-resources` 返回 200；
    - 学生访问同接口返回 403；
    - 管理员访问维持 200。
  - Why：避免后续回归为仅管理员可访问，明确“教师可访问”的契约。
  - How：复用现有登录流程与请求头注入方式，按角色分别发起请求并断言状态码。
- 前端无需改动（仅回归验证）
  - 文件：`aigc-site-react/src/pages/MediaResourcesPage.tsx`
  - What：不改代码，只验证教师登录后不再触发 403 toast，列表可正常渲染。
  - Why：前端权限配置已正确，问题根因在后端。

## Assumptions & Decisions
- 决策：媒体资源目录权限采用“教师/管理员可读，学生不可读”。
- 假设：当前 `RequestUserContext.canManageTeaching()` 对教师返回 `true`，对学生返回 `false`，可直接复用 `requireTeachingManager()`。
- 决策：本次不调整路由路径 `/api/v1/admin/media-resources` 与前端接口路径，避免扩大变更范围。
- 决策：不触及模型配置、服务商中心权限（继续仅管理员）。

## Verification Steps
- 后端自动化验证
  - 运行 `AuthIntegrationTest` 中媒体资源相关用例，确认管理员/教师 200、学生 403。
- 前端联调验证
  - 教师账号进入媒体资源页面，确认不再出现“只有管理员可以查看媒体资源目录”报错，数据可展示。
  - 学生账号进入媒体资源页面，确认仍显示无权限提示。
- 回归检查
  - 管理员账号访问媒体资源页面保持正常。
  - 模型配置与服务商中心仍仅管理员可见/可访问。
