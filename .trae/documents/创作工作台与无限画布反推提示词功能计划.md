# 创作工作台与无限画布“反推提示词”功能实施计划

## Summary
- 目标：新增“反推提示词”能力，支持用户上传/粘贴图片后调用**已配置豆包系列模型**反推提示词，并在“创作工作台”“无限画布”均可进入使用。
- 入口策略：采用**独立入口页**（`/tools/reverse-prompt`），工作台与画布通过显式入口跳转到同一页面。
- 模型策略：前端可选模型；后端仅返回并允许调用“模型名包含 `doubao` 或 `seedream`”的可用模型。
- 输出策略：返回结构化结果（正向提示词、反向提示词、风格、镜头、光线、构图、色彩、参数建议）。
- 存储策略：本期仅做即时结果，不写入现有 `/history` 任务链路。

## Current State Analysis
- 前端工作台主体在 `aigc-site-react/src/pages/WorkspacePage.tsx`，能力输入面板在 `aigc-site-react/src/components/workspace/PromptPanel.tsx`。
- 无限画布页在 `aigc-site-react/src/pages/CanvasPage.tsx`，核心组件在 `aigc-site-react/src/components/canvas/ComfyLikeCanvas.tsx`。
- 路由集中在 `aigc-site-react/src/router.tsx`，`WorkspaceRouteVariant` 定义在 `aigc-site-react/src/routes/types.ts`。
- 现有图片上传+Base64 转换组件可复用：`aigc-site-react/src/components/workspace/VideoReferenceImageField.tsx`。
- 后端生成入口为 `aigc-server/src/main/java/com/example/aigc/controller/GenerationController.java` 与 `GenerationServiceImpl`，当前仅覆盖 `text/image/both/video`。
- 后端已有统一模型连接与网关调用能力：`ModelConfigRepository` + `ModelCapabilityService` + `ProviderHttpGateway` + `ProviderCatalog`，可直接复用完成多模态请求转发。

## Proposed Changes

### 1) 后端：新增“反推提示词”独立 API（不侵入 generate 历史链路）
- 新增 DTO：
  - `aigc-server/src/main/java/com/example/aigc/dto/ReversePromptRequest.java`
  - `aigc-server/src/main/java/com/example/aigc/dto/ReversePromptResponse.java`
  - `aigc-server/src/main/java/com/example/aigc/dto/ReversePromptModelOptionsData.java`
- 新增服务接口与实现：
  - `aigc-server/src/main/java/com/example/aigc/service/ReversePromptService.java`
  - `aigc-server/src/main/java/com/example/aigc/service/impl/ReversePromptServiceImpl.java`
- 新增控制器：
  - `aigc-server/src/main/java/com/example/aigc/controller/ReversePromptController.java`
  - 提供：
    - `GET /api/v1/reverse-prompt/models`：返回可选豆包模型（仅 `doubao|seedream`）
    - `POST /api/v1/reverse-prompt/generate`：输入图片（URL 或 data:image Base64）+ 可选模型，输出结构化反推结果
- 复用与约束：
  - 鉴权复用 `RequestAuthService.requireUserId(...)`。
  - 模型解析复用 `ModelConfigRepository` + `ConnectionConfigRepository` + `ModelCapabilityService`。
  - 外部调用复用 `ProviderHttpGateway.invokeChat(...)`，按 OpenAI 兼容多模态消息组织内容。
  - 仅允许调用“模型名含 `doubao`/`seedream`”且连接可用的模型；若用户传模型不在允许列表则 400。
  - 支持图片输入：`http(s)` 与 `data:image/...;base64,...`；增加体积与格式校验（沿用工作台现有上限思路，服务端兜底）。
  - 输出统一 JSON 结构，若模型返回纯文本则做解析降级（尽量抽取字段，兜底落 `positivePrompt`）。

### 2) 前端：新增独立“反推提示词”页面

  - `aigc-site-react/src/pages/ReversePromptPage.tsx`
  - `aigc-site-react/src/components/workspace/ReversePromptPanel.tsx`（可与页面同文件，视当前代码风格决定）
- 新增 API 与类型：
  - `aigc-site-react/src/types/index.ts`：新增 `ReversePromptRequest/Response/ModelOptions` 类型。
  - `aigc-site-react/src/api/index.ts`：新增
    - `getReversePromptModels()`
    - `generateReversePrompt()`
- 交互形态：
  - 图片输入复用 `VideoReferenceImageField`（上传/拖拽/URL/Base64）。
  - 模型下拉来自 `GET /reverse-prompt/models`，支持用户手选。
  - 结果区展示结构化字段，并提供“一键复制正向”“一键复制反向”“复制完整 JSON/Markdown”。
  - 错误态沿用现有 toast + 页面错误文案模式。

## Assumptions & Decisions
- 已确认：独立入口页方案；工作台与画布均通过跳转进入。
- 已确认：模型由用户选择，候选范围为“已配置且可用且模型名命中 `doubao|seedream`”。
- 已确认：输出为结构化“正向+反向+参数建议”。
- 已确认：图片输入同时支持上传与 URL。
- 已确认：本期不写入 `/history`，仅即时显示结果。
- 约定：若某环境下无可用豆包模型，前端展示空态引导去“模型配置/服务商中心”完成配置。

## Verification Steps
- 后端接口验证：
  - `GET /api/v1/reverse-prompt/models` 返回仅豆包候选模型。
  - `POST /api/v1/reverse-prompt/generate` 在 URL 图与 Base64 图两种输入下均可返回结构化结果。
  - 非豆包模型ID提交返回 400。
  - 未登录或鉴权缺失返回 401/403（与现有鉴权策略一致）。
- 前端流程验证：
  - `/tools/reverse-prompt` 可进入、可上传图片、可切换模型、可出结果并复制。
  - “创作工作台”标签可进入反推页。
  - “无限画布”快捷入口与工具栏按钮可跳转反推页。
- 回归验证：
  - 原有 `/workspace` 文生图/文生视频/图生视频流程不受影响。
  - `CanvasPage` 与 `ComfyLikeCanvas` 的现有保存/提交 Comfy 逻辑不受影响。
  - 关键前后端测试通过，且新增代码无明显诊断报错。
