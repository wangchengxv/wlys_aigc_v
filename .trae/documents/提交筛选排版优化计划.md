# 作业提交筛选功能排版优化计划

## 问题分析

当前"提交筛选"区域的排版存在以下问题：

### 现有结构
1. **第一行布局固定为3列**：`grid-template-columns: repeat(3, minmax(180px, 1fr))`
2. **教师端筛选条件**：
   - 第一行：关键词 + 提交状态 + 班级（3个）
   - 第二行：提交人 + 重置按钮（2个元素，布局不平衡）
3. **学生端筛选条件**：
   - 第一行：关键词 + 提交状态（2个，第三个格子空置浪费空间）
4. **问题**：
   - 学生端明明只有2个筛选条件，却占用3列空间，导致元素过宽且分散
   - 教师端第二行只有1个筛选条件，与第一行3列不对齐
   - 重置按钮单独一行，显得突兀

## 优化方案

### 方案：使用自动填充网格 + 自适应列宽

修改 `components.css` 中的 `.teaching-form__row` 样式：

```css
.teaching-form__row {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(180px, 1fr));
  gap: 12px;
  align-items: end;
}
```

### 具体改动

1. **components.css** - 修改 `.teaching-form__row` 样式
   - 将 `repeat(3, minmax(180px, 1fr))` 改为 `repeat(auto-fit, minmax(180px, 1fr))`
   - 这样元素会根据实际数量自动分配空间
   - 2个元素会平分一行，3个元素会平分一行

2. **TeachingAssignmentDetailPage.tsx** - 调整布局结构
   - 将"提交人"筛选器和"重置筛选"按钮合并到同一行
   - 重置按钮可使用 `<div className="teaching-actions">` 或直接放在筛选器后面

## 修改文件清单

| 文件 | 修改内容 |
|------|---------|
| `aigc-site-react/src/styles/components.css` | 修改 `.teaching-form__row` 样式为自动填充网格 |
| `aigc-site-react/src/pages/TeachingAssignmentDetailPage.tsx` | 调整筛选器HTML结构，将提交人和重置按钮放在同一行 |

## 预期效果

- **学生端**：关键词和状态筛选器平分一行，美观大方
- **教师端**：第一行3个筛选器平分，第二行提交人和重置按钮平分
- **整体**：布局更加平衡紧凑，视觉层次清晰
