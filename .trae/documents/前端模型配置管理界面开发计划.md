# 前端模型配置管理界面开发计划

## 任务概述
为 aigc-site 前端添加模型配置管理界面，使用户可以通过网页进行连接配置和模型配置的增删改查操作。

## 需求分析

### 后端 API（已实现）
- **连接配置 API** (`/api/v1/connections`)
  - `POST /api/v1/connections` - 创建连接配置
  - `GET /api/v1/connections` - 获取所有连接配置
  - `GET /api/v1/connections/{id}` - 获取单个连接配置
  - `PUT /api/v1/connections/{id}` - 更新连接配置
  - `DELETE /api/v1/connections/{id}` - 删除连接配置

- **模型配置 API** (`/api/v1/models`)
  - `POST /api/v1/models` - 创建模型配置
  - `GET /api/v1/models` - 获取所有模型配置
  - `GET /api/v1/models/{id}` - 获取单个模型配置
  - `PUT /api/v1/models/{id}` - 更新模型配置
  - `DELETE /api/v1/models/{id}` - 删除模型配置

## 实施步骤

### 第一步：添加类型定义
1. 在 `types/index.ts` 中添加连接配置和模型配置的类型定义
   - `ConnectionConfig`
   - `ModelConfig`
   - 请求/响应类型

### 第二步：添加 API 服务函数
1. 在 `services/api.ts` 中添加：
   - `getConnections()` - 获取连接列表
   - `createConnection(data)` - 创建连接
   - `updateConnection(id, data)` - 更新连接
   - `deleteConnection(id)` - 删除连接
   - `getModels()` - 获取模型列表
   - `createModel(data)` - 创建模型
   - `updateModel(id, data)` - 更新模型
   - `deleteModel(id)` - 删除模型

### 第三步：创建模型配置管理视图
1. 新建 `views/ModelConfigView.vue`
   - Tab 切换：连接配置 / 模型配置
   - 连接配置列表展示（表格形式）
   - 模型配置列表展示（表格形式）
   - 创建/编辑的表单弹窗
   - 删除确认弹窗

### 第四步：添加路由
1. 在 `router/index.ts` 中添加路由
   - `/models` -> ModelConfigView

### 第五步：更新导航入口
1. 在 `App.vue` 或现有布局中添加导航入口
   - 在设置页面添加跳转链接，或添加独立导航

## UI 设计

### 布局结构
```
/models
├── Tab导航: [连接配置] [模型配置]
├── 搜索/筛选区域 (可选)
├── 数据表格
│   ├── 连接配置Tab列: 名称 | 提供商 | Base URL | API Key(掩码) | 状态 | 操作
│   └── 模型配置Tab列: 名称 | 提供商 | 模型标识 | 关联连接 | 状态 | 操作
└── 底部操作栏: [新建] 按钮
```

### 表单设计
#### 连接配置表单
- 名称 (input, 必填)
- 提供商 (input, 必填)
- Base URL (input, 必填)
- API Key (input password, 必填)
- 启用状态 (switch/toggle)

#### 模型配置表单
- 名称 (input, 必填)
- 提供商 (input, 必填)
- 模型标识 (input, 必填)
- 关联连接 (select, 下拉选择已保存的连接)
- 启用状态 (switch/toggle)
- 元数据 (可选，JSON editor 或 key-value pairs)

## 组件复用
- 使用现有组件: `AppButton`, `AppInput`, `TagSelector`
- 新增通用组件（按需）:
  - `ConfirmDialog.vue` - 确认对话框
  - `DataTable.vue` - 通用数据表格
  - `FormDialog.vue` - 通用表单弹窗
