# 流程工作区放大与顶部对齐计划

## Summary
- 目标：放大“流程工作区”主内容区域，使其与上方页面标题工具栏宽度对齐，消除左右明显留白。
- 范围：按已确认方案，统一作用于全部流程页（剧本与故事、场景与道具、导演模式、配音与旁白、成片与导出）。
- 实现方式：移除流程页容器的固定最大宽度限制，并让容器在父布局中占满可用宽度。

## Current State Analysis
- 布局主容器 `app-shell__content` 已是宽容器，当前配置为 `width: 100%` 且 `max-width: 1520px`，可承载更宽内容。
- 流程页公共组件 `WorkflowScriptProjectHub` 使用类名 `workflow-sub-page workflow-sub-page--scenes` 作为页面根容器。
- 在 `aigc-site-react/src/styles/components.css` 中，`.workflow-sub-page--scenes` 被限制为：
  - `max-width: 1080px;`
  - `margin-inline: auto;`
- 该限制会导致流程页内容宽度显著小于上方工具栏视觉宽度，形成截图中的两侧空白。

## Proposed Changes
- 文件：`/Users/xingyi/Downloads/AIGC_小云雀/AIGC_university_副本/aigc-site-react/src/styles/components.css`
- 改动点：`.workflow-sub-page--scenes` 规则
- 具体修改：
  - 删除 `max-width: 1080px;`（去除固定窄容器约束）。
  - 将居中约束由固定宽度居中调整为全宽占位（保留正常流式布局），确保视觉与上方窗口同宽体系。
  - 如需最小侵入，优先采用 `width: 100%; max-width: none; margin-inline: 0;` 或等效写法，只改该选择器，不影响其它模块样式。
- 影响说明：
  - `WorkflowScriptProjectHub` 被多个流程页复用，本次修改可一次性统一所有流程页宽度表现，符合“全流程页统一”决策。
  - 文案行宽受 `workflow-sub-page__lede` 的 `max-width: 42rem` 保护，不会因容器变宽导致段落过长难读。

## Assumptions & Decisions
- 决策已锁定：改动范围为全流程页统一，不做单页特判。
- 假设：上方“窗口对齐”的参照为同页顶部工具栏/标题区域的内容宽度，而非浏览器全屏贴边。
- 约束：仅做样式层改动，不改 React 组件结构与路由逻辑。

## Verification Steps
- 进入任一流程页（例如“流程工作区 > 剧本与故事”）检查：
  - 主内容卡片组左右边界与上方标题工具栏边界对齐。
  - 原截图中的两侧大块留白消失或显著缩小。
- 横向验证其它流程页（场景与道具、导演模式、配音与旁白、成片与导出）：
  - 宽度策略一致，无单页回退到窄容器。
- 响应式验证：
  - 在 `<= 960px` 与 `<= 1180px` 断点下，页面仍按既有媒体查询折叠，无横向滚动异常。
- 回归检查：
  - 非流程页（如首页、课程页、模型页）布局不应受到影响。
