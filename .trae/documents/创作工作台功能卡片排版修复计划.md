# 创作工作台功能卡片排版修复计划

## Summary
- 目标：修复“创作工作台”中功能卡片（输入与参数 / 结果与下载）的错误排版，消除左侧挤压与右侧大面积留白，恢复一致、可读、可操作的布局。
- 核心策略：对齐 `FixedPanelDock` 现有组件结构与样式实现，移除/覆盖遗留的旧版两栏规则，并优化工作台卡片操作区在桌面与窄屏下的对齐与换行。

## Current State Analysis
- 当前页面入口与组件链路：
  - `aigc-site-react/src/pages/WorkspacePage.tsx` 使用 `FixedPanelDock` 承载 `PromptPanel` 与 `ResultPanel`。
  - `aigc-site-react/src/components/common/FixedPanelDock.tsx` 当前 DOM 结构是：`head + tabs + panel`（不再有旧版 `rail` 容器）。
- 已定位的样式冲突事实：
  - `aigc-site-react/src/styles/components.css` 中仍存在旧版 `.fixed-panel-dock` 两列布局规则：`grid-template-columns: minmax(240px, 280px) minmax(0, 1fr);`，并配套 `.fixed-panel-dock__rail*` 规则。
  - 该规则与当前 `FixedPanelDock` 结构不匹配，会把子节点自动放入两列网格，导致主面板落入窄列，出现“左窄右空”的视觉问题（与截图一致）。
  - 同时，`PromptPanel` 的 `.bottom-actions` 当前是 `justify-content: space-between`，在宽度异常或临界宽度下会拉大主按钮组与“AI视频生成”按钮间距，进一步放大排版错位感。

## Proposed Changes

### 1) 修复 Dock 主布局与组件结构一致性
- 文件：`aigc-site-react/src/styles/components.css`
- 变更内容：
  - 移除或禁用旧版 `.fixed-panel-dock` 两列布局定义（及无效的 `__rail` 依赖规则影响）。
  - 将 `.fixed-panel-dock` 对齐为单列流式布局（与 `global.css` 中 revamp 规则一致）：头部、tabs、panel 纵向排列，主面板宽度占满可用空间。
  - 保留并复用有效的 `__panel`、`__panel-body`、`__tabs` 风格，避免影响当前视觉语言。
- 预期收益：
  - 功能卡片恢复正常宽度，不再被压缩到左侧窄栏。
  - 工作台主内容区利用率恢复，右侧异常留白消失。

### 2) 调整 Prompt 卡片底部操作区的对齐策略
- 文件：`aigc-site-react/src/styles/components.css`
- 变更内容：
  - 优化 `.prompt .bottom-actions` 与 `.prompt .feature-actions*` 在桌面端的对齐方式，避免极端“左右拉扯”。
  - 为中等宽度与窄屏补充明确换行策略，保证按钮组在同一视觉块内连续展示，不出现漂移到远侧的孤立按钮。
  - 保持现有交互语义不变（开始生成 / 再来一版 / 清空 / AI视频生成）。
- 预期收益：
  - 功能按钮分组更紧凑，阅读与点击路径更稳定。
  - 在 `/workspace` 与 `/tools/*` 各入口中保持一致排版体验。

### 3) 最小化代码调整（如需要）
- 文件：`aigc-site-react/src/pages/WorkspacePage.tsx`（仅在必要时）
- 变更内容：
  - 如需更精细样式隔离，可给 `FixedPanelDock` 追加页面级 className（例如 `workspace-page__dock`）用于定向覆盖。
  - 默认优先通过样式修复，不改动业务逻辑与状态流。
- 预期收益：
  - 将布局修复限定在工作台上下文，降低对其他复用场景的影响风险。

## Assumptions & Decisions
- 决策 1：本次只修复布局与视觉对齐，不调整生成逻辑、API 调用、状态管理。
- 决策 2：优先清理 `components.css` 中与当前组件不匹配的遗留规则，而非“继续叠加覆盖”，以减少后续维护成本。
- 决策 3：保持 `WorkspacePage` 当前信息架构（统计条、模式切换、Dock 面板、帮助提示）不变，仅处理卡片排版问题。

## Verification Steps
- 代码级验证：
  - 检查 `FixedPanelDock` 样式最终生效链路，确认不存在将主面板放入窄列的网格规则。
  - 检查 `PromptPanel` 操作区在桌面/窄屏断点下的 flex/grid 行为是否符合预期。
- 构建与静态检查：
  - 在 `aigc-site-react` 执行 `npm run lint`。
  - 在 `aigc-site-react` 执行 `npm run build`。
- 页面回归（手动）：
  - 路由：`/workspace`、`/tools/image`、`/tools/video`、`/tools/image-to-video`。
  - 验收点：
    - 功能卡片宽度正常，占据主内容区，不再左窄右空。
    - “开始生成 / 再来一版 / 清空 / AI视频生成”对齐合理，无异常漂移。
    - 切换 “输入与参数 / 结果与下载” 标签时面板布局稳定。
