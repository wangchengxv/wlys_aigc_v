# 全局右侧抽屉加宽至720计划

## Summary
- 目标：解决右侧抽屉内容拥挤、文本挤在一起难辨析的问题。
- 范围：统一加宽全站 `ActionDrawer`（所有右侧抽屉），桌面目标宽度约 `720px`。
- 原则：仅调整抽屉容器宽度，不改业务表单逻辑，移动端仍保持自适应不溢出。

## Current State Analysis
- 右侧抽屉组件由 `aigc-site-react/src/components/common/ActionDrawer.tsx` 统一承载，面板类名为 `action-drawer__panel`。
- 当前宽度样式定义在 `aigc-site-react/src/styles/global.css`：`.action-drawer__panel { width: min(520px, 100%); }`。
- 页面复用面广：如 `ScriptProjectCreatePage`、`TeachingCoursesPage`、`TeachingCourseDetailPage`、`AdminDirectoryPage` 等均通过 `ActionDrawer` 打开右侧抽屉。
- 因为是统一组件，改这一处即可全局生效，且不会影响左侧主导航 `app-shell-nav`。

## Proposed Changes
- 文件：`aigc-site-react/src/styles/global.css`
- 修改点：将 `.action-drawer__panel` 的宽度从 `min(520px, 100%)` 调整为 `min(720px, 100%)`。
- 修改原因：在桌面端直接扩大可用排版空间，缓解选项文字、提示文案、输入区并列时的拥挤和换行堆叠问题。
- 修改方式：只变更该单一宽度值，保留 `100%` 上限以确保窄屏不超出视口。

## Assumptions & Decisions
- 决策已锁定：按你的选择，统一加宽“所有右侧抽屉”，目标宽度 `720px`。
- 假设：当前主要问题来自容器宽度不足，不优先改动各页面表单网格（如双列改单列）。
- 兼容策略：移动端由于 `min(720px, 100%)` 的 `100%` 上限，仍按可视宽度收缩，不引入横向滚动。

## Verification Steps
- 启动前端并进入至少 4 个使用抽屉的页面验证：
  - `新建剧本项目` 抽屉
  - `教学课程` 新建/编辑抽屉
  - `课程详情` 相关抽屉
  - `组织用户` 页面抽屉
- 核验项：
  - 抽屉在桌面端明显变宽（接近 720px）。
  - 表单文本可读性提升，关键文案不再密集挤压。
  - 抽屉开关、滚动、关闭按钮行为正常。
  - 小屏下抽屉不溢出视口、无横向滚动。
- 回归检查：
  - 左侧主导航宽度与主内容区布局不受影响。
  - 其他非抽屉弹层（Modal/页面主体）样式不受影响。
