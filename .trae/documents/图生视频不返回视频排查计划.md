# 图生视频功能不返回视频的排查计划

## 问题概述
图生视频功能已调用成功并完成扣费，但未返回视频结果。

## 排查步骤

### 1. 分析代码流程
检查 `callKlingVideoApi` (第 1136-1179 行) 的调用流程：
- 提交请求到 `/kling/v1/videos/image2video`
- 调用 `extractKlingTaskResultUrls` 尝试直接提取视频 URL
- 如果没有直接返回，解析 `task_id` 并轮询查询状态

### 2. 检查响应解析逻辑

#### 2.1 任务 ID 解析 (`parseArkVideoTaskId`, 第 2555-2586 行)
```java
// 检查字段顺序：
// 1. body.task_id
// 2. body.data.task_id
// 3. body.data[0].task_id (列表中第一个元素)
```
**潜在问题**：Kling API 返回的 task_id 可能位于其他嵌套路径

#### 2.2 视频 URL 提取 (`extractKlingTaskResultUrls`, 第 1233-1252 行)
```java
// 期望的数据结构：
// data -> task_result -> videos -> [{url: "...", id: "..."}]
```
**潜在问题**：
- `videos` 字段可能为空或字段名不同
- URL 可能嵌套在更深层或不同的字段中

#### 2.3 URL 收集逻辑 (`collectCandidateUrls`, 第 2511-2552 行)
```java
// 只收集包含 "url" 关键字的字段值
if (key.contains("url")) {
    String maybe = valueAsString(value);
    if (isValidMediaUrl(maybe)) {
        urls.add(maybe);
    }
}
```
**潜在问题**：如果 Kling 返回的字段名不包含 "url"（如 `video_url` 或 `file`），可能无法正确提取

### 3. 状态判断逻辑检查

#### 3.1 任务状态解析 (`parseArkTaskStatus`, 第 2588-2604 行)
```java
// 检查字段顺序：
// body.status, body.task_status, body.state
// body.data.status, body.data.task_status
```
**潜在问题**：Kling 可能使用不同的状态字段名

#### 3.2 成功状态判断 (`isSuccessStatus`, 第 2623-2634 行)
```java
// 支持的成功状态：success, succeeded, done, completed, finish, finished
```
**潜在问题**：如果 Kling 返回的状态值不匹配，轮询会超时

### 4. 轮询配置检查

#### 4.1 轮询参数 (`pollKlingTaskForUrls`, 第 1181-1231 行)
- `maxAttempts`: 由 `arkProperties.getVideoPollMaxAttempts()` 配置
- `intervalMs`: 由 `arkProperties.getVideoPollIntervalMs()` 配置
- **潜在问题**：轮询次数不足或间隔太长

### 5. 修复方案

#### 方案 A：增强 `collectCandidateUrls` 的字段匹配
```java
// 扩展关键字匹配
if (key.contains("url") || key.equals("video") || key.equals("file")) {
    // ... 处理逻辑
}
```

#### 方案 B：添加 Kling 特定的响应解析
在 `extractKlingTaskResultUrls` 中针对 Kling API 的响应格式进行特殊处理：
```java
// 检查 task_result.videos[].url
// 检查 task_result.videos[].output_url
// 检查 task_result.videos[].download_url
```

#### 方案 C：增加调试日志
在关键路径添加日志输出完整响应结构，便于排查
