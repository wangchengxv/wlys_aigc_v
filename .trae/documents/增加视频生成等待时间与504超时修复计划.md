# 增加视频生成等待时间与 504 超时修复计划

## Summary
- 目标：解决 `code: 504, message: "视频生成超时，请稍后重试或缩短提示词"`，将视频轮询等待窗口从当前约 2 分钟提升到约 5 分钟。
- 范围：按你的选择，统一作用于全部视频通道（Ark/OneLink/Vidu/Kling/Moark）当前共用的轮询逻辑。
- 成功标准：
  - 默认轮询总等待时间提升到约 300 秒。
  - 不改变现有 API 协议与错误码结构。
  - 可通过配置项继续按环境调整等待时长。

## Current State Analysis
- 超时报错来源位于 `aigc-server/src/main/java/com/example/aigc/service/impl/GenerationServiceImpl.java`：
  - `pollArkVideoTask(...)` 最终抛出 `视频生成超时，请稍后重试或缩短提示词`。
  - `pollVideoTaskByPath(...)` 也使用同样超时文案与同样轮询参数。
- 当前轮询参数来自 `AigcArkProperties`：
  - `videoPollMaxAttempts = 40`
  - `videoPollIntervalMs = 3000`
  - 总等待窗口约 `40 * 3s = 120s`。
- 配置映射在 `aigc-server/src/main/resources/application.yml`：
  - `aigc.ark.video-poll-max-attempts: 40`
  - `aigc.ark.video-poll-interval-ms: 3000`
- 结论：这是全局视频轮询参数，调高后会同时影响所有走这两条轮询路径的视频通道，符合“全部视频通道”诉求。

## Proposed Changes

### 1) 提升默认轮询窗口到 5 分钟
- 文件：`aigc-server/src/main/java/com/example/aigc/config/AigcArkProperties.java`
- 变更：
  - 将默认 `videoPollMaxAttempts` 从 `40` 提升到 `100`（`interval` 保持 `3000ms`）。
- 原因：
  - 以最小侵入方式达到约 300 秒等待窗口，不改调用链。

### 2) 同步应用层默认配置
- 文件：`aigc-server/src/main/resources/application.yml`
- 变更：
  - 将 `aigc.ark.video-poll-max-attempts` 从 `40` 提升到 `100`。
  - 保留 `aigc.ark.video-poll-interval-ms: 3000` 不变。
- 原因：
  - 保证本地与未显式注入环境变量时的行为一致，避免“代码默认值与配置值不一致”。

### 3) 保持统一轮询逻辑不分叉
- 文件：`aigc-server/src/main/java/com/example/aigc/service/impl/GenerationServiceImpl.java`
- 变更：
  - 不新增 provider 分支，不改业务协议；继续复用 `pollArkVideoTask(...)` 与 `pollVideoTaskByPath(...)` 的统一参数读取。
- 原因：
  - 你选择“全部视频通道”统一生效，当前架构已天然满足，避免不必要复杂度。

## Assumptions & Decisions
- 已确认决策：
  - 等待策略：5 分钟。
  - 影响范围：全部视频通道。
- 假设：
  - 当前超时主要由轮询窗口不足导致，而非任务必然失败或回调异常。
  - 环境允许请求在最长约 5 分钟后再返回最终超时。
- 不在本次范围：
  - 不新增异步回调/消息队列改造。
  - 不修改错误码与前端提示文案。
  - 不改变单次 HTTP 轮询请求超时（`Duration.ofSeconds(30)`）的实现。

## Verification Steps
- 配置验证：
  - 启动后确认 `aigc.ark.video-poll-max-attempts` 生效为 `100`（默认环境）。
- 业务验证：
  - 用容易超过 2 分钟的视频任务复测，确认不再在约 120 秒即返回 `504`。
  - 在约 300 秒窗口内观察任务是否成功回写 URL；若仍超时，才返回同样 `504`。
- 回归验证：
  - 复测 Ark/OneLink/Vidu/Kling/Moark 任意各 1 条视频任务，确保统一轮询逻辑无回归。
