# 项目详情页面排版优化计划

## 1. 问题分析

当前 `ScriptProjectDetailPage.tsx` 存在以下排版问题：

1. **Hero 区域三栏布局过密**：`delivery-hero.panel` 使用三列网格（封面 / 内容 / 侧边KPI），在中等宽度屏幕下容易挤压
2. **内容区信息过密**：
   - 状态标签行 (`delivery-status-row`) 塞入 3 个标签 + 审核意见说明，视觉超载
   - 描述列表 (`delivery-description-list`) 采用 2 列密集排列，每项内容偏小
   - KPI 卡片 (`delivery-kpi-card`) 数量多（4个），尺寸偏小
3. **FixedPanelDock 展开区过挤**：每个面板内内容块堆叠过密，间距不足
4. **PipelineProgressBar 与 Dock 间距未预留**

## 2. 优化方向

### 2.1 Hero 区域重构
- 将三栏布局改为更宽松的两栏布局：左（封面）+ 右（内容与 KPI 合并）
- 封面比例调大，留出呼吸感
- 内容区顶部（标题/摘要）与底部（状态标签）之间增加间距

### 2.2 状态标签区精简
- 标签保留项目阶段、审核状态、交付状态三个核心标识
- 审核意见改为折叠/展开，默认只显示摘要

### 2.3 信息卡间距优化
- `delivery-section-card` 内部间距从 `var(--space-lg)` 增加到 `var(--space-xl)`
- `delivery-description-list` 从 2 列改为单列，减少每个条目宽度
- KPI 卡片改为一排 4 个或 2x2，增大单个卡片尺寸

### 2.4 Dock 面板间距优化
- 各面板之间增加 `gap: var(--space-xl)`
- 面板内部元素间距优化，避免堆砌感

## 4. 实施步骤

1. **CSS 优化**（components.css）
   - 调整 `.delivery-hero.panel` 布局为两栏
   - 增大 `.delivery-section-card` 内部间距
   - 优化 `.delivery-description-list` 为单列
   - 增大 `.delivery-kpi-grid--compact` 卡片尺寸
   - 优化 `.delivery-status-row` 间距
   - 调整 `.fixed-panel-dock__panel` 间距

2. **JSX 调整**（ScriptProjectDetailPage.tsx）
   - 调整 Hero 区域 DOM 结构，适配新 CSS
   - 精简状态标签行显示
   - 审核意见改为折叠显示

## 5. 验收标准

- 页面在 1280px 宽度下布局宽松，不拥挤
- 各信息块间距一致，有呼吸感
- 核心信息（项目名、状态、入口）突出
