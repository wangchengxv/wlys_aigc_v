# 全站小字备注与英文标题收敛计划

## Summary
- 目标：全站隐藏“工作区功能说明/小字备注”等次级文案，默认不展示；仅在鼠标悬停主标题 800ms 后显示对应备注；同时去掉所有英文小标题文案。
- 结果：界面首屏更干净，信息层级更聚焦在中文主标题，细节说明改为按需显隐。

## Current State Analysis
- 已确认当前英文小标题主要来源：
  - 路由元数据 `handle.eyebrow`（如 `Workspace`、`Tools`、`Overview`）在 `PageToolbar` 展示。
  - `WorkspacePage`、`FixedPanelDock`、部分项目页中硬编码英文眉标题（如 `Workspace Modes`、`Workspace Panels`、`Create`、`Result`）。
- 已确认“小字备注”主要来源：
  - `PageToolbar.description`、大量 `.muted` 段落、`small`、部分 `summary`/`hint` 文案。
  - 这些文案广泛分布于页面和通用组件中，直接逐页硬删维护成本高且易漏。
- 现状结构适合“统一机制改造”：
  - 站点由 `AppLayout` + 公共组件（`PageToolbar`、`FixedPanelDock`、`ProjectSubpageShell` 等）承载大部分标题与备注展示。
  - 可通过“统一 class + 轻量交互组件”实现全站规则，避免大量逐页业务逻辑改造。

## Proposed Changes

### 1) 建立全站“延时悬停显示备注”机制（800ms）
- 文件：
  - `aigc-site-react/src/components/common/PageToolbar.tsx`
  - `aigc-site-react/src/components/common/FixedPanelDock.tsx`
  - `aigc-site-react/src/components/script/ProjectSubpageShell.tsx`
  - `aigc-site-react/src/styles/global.css`
  - `aigc-site-react/src/styles/components.css`
- 变更内容：
  - 在上述标题容器中增加统一语义结构：主标题元素 + 备注元素（默认隐藏）。
  - 通过 CSS + 延时交互类（hover/focus-within + transition-delay 800ms）控制备注显示。
  - 仅鼠标停留超过 800ms 显示；移出后隐藏。
- 原因：
  - 满足“默认隐藏小字备注、长时间悬停才展示”的全站规则。
- 实施方式：
  - 优先采用样式层统一规则，必要时用最小组件状态控制（避免逐页重复代码）。

### 2) 全站去英文小标题（UI 文案层）
- 文件：
  - `aigc-site-react/src/layouts/AppLayout.tsx`
  - `aigc-site-react/src/router.tsx`
  - `aigc-site-react/src/pages/WorkspacePage.tsx`
  - `aigc-site-react/src/components/common/FixedPanelDock.tsx`
  - `aigc-site-react/src/components/script/ProjectSubpageShell.tsx`
  - `aigc-site-react/src/styles/global.css`
  - `aigc-site-react/src/styles/components.css`
- 变更内容：
  - 移除或替换所有英文眉标题文案（如 `Workspace`、`Create`、`Result`、`Tools`、`Overview` 等）为中文，或直接隐藏眉标题层。
  - `router.tsx` 中 `handle.eyebrow` 统一中文化/置空策略，确保 `PageToolbar` 不再出现英文。
  - 对历史页面中硬编码英文标题做统一清理。
- 原因：
  - 满足“去掉英文标题”并保持全站一致。

### 3) “工作区这里的功能”默认隐藏（按你截图语义）
- 文件：
  - `aigc-site-react/src/layouts/AppLayout.tsx`
  - `aigc-site-react/src/styles/global.css`
- 变更内容：
  - 顶部“工作区”下方次级功能字样（当前是 `handle.section` 类型的小字）默认隐藏。
  - 同样套用 800ms 悬停在“工作区”大字上再显示。
- 原因：
  - 与你截图中红箭头指向区域保持一致规则。

### 4) 全站备注类文案的统一收口规则
- 文件：
  - `aigc-site-react/src/styles/global.css`
  - `aigc-site-react/src/styles/components.css`
  - 重点页面：
    - `aigc-site-react/src/pages/WorkspacePage.tsx`
    - `aigc-site-react/src/pages/HomePage.tsx`
    - `aigc-site-react/src/pages/SettingsPage.tsx`
    - `aigc-site-react/src/pages/MediaResourcesPage.tsx`
    - `aigc-site-react/src/pages/ScriptProject*.tsx`
- 变更内容：
  - 对统一识别的小字备注类（`.muted`、`.page-toolbar__description`、局部 `small`/`hint`）引入“默认隐藏 + 标题延时显示”的容器规则。
  - 对必须保留常显的状态提示（错误、权限阻断、表单校验）建立白名单 class（如 `.always-visible-note`）避免误隐藏。
- 原因：
  - 兼顾“全站统一”与“关键可用性信息不丢失”。

## Assumptions & Decisions
- 范围决策：按“全站统一”执行，不仅限工作台。
- 交互决策：悬停显示延时固定 800ms。
- 文案决策：所有英文小标题在 UI 层移除或替换为中文，不保留英文眉标题。
- 可用性决策：错误提示/权限提示/阻断提示保持可见，不纳入默认隐藏。
- 兼容性决策：桌面端使用 hover 延时；无鼠标设备不强制触发延时显示（保留必要信息可读性）。

- 静态检查：
  - 全局检索英文眉标题关键字（`Workspace|Create|Result|Tools|Overview|Settings|Governance|Auth|Error`）应不再出现在界面标题位置。
  - 检查小字备注 class 是否都受统一显隐规则管理。
- 构建验证：
  - `aigc-site-react` 执行 `npm run lint`
  - `aigc-site-react` 执行 `npm run build`
- 人工回归（桌面）：
  - 路由抽查：`/`、`/workspace`、`/tools/image`、`/courses`、`/script-projects`、`/settings`。
  - 验收点：
    - 小字备注默认隐藏；
    - 鼠标停留主标题超过 800ms 后出现；
    - 所有英文小标题消失；
    - 错误/权限类提示仍可见；
    - 不影响主要操作按钮点击与流程。
