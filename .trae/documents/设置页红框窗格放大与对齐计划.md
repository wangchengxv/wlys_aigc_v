# 设置页红框窗格放大与对齐计划

## Summary
- 目标：将 `SettingsPage` 红框内全部窗格（第一排双卡 + 环境信息 + 第三方账号绑定）整体放大，并在视觉上与上方 `StatStrip` 区域实现更强的左右与体量对齐。
- 范围：仅修改设置页专属样式，不影响全站其他 `content-card` 使用页面。
- 结果预期：红框内卡片宽度利用率更高、内容区留白更饱满、第一排两张卡高度协调，整体视觉重心与顶部统计卡一致。

## Current State Analysis
- 页面结构位于 `src/pages/SettingsPage.tsx`：
  - 顶部是 `StatStrip`（三项统计卡）。
  - 红框第一排由 `.content-card-grid` 承载两张 `.content-card`。
  - 红框后续两张卡同样使用 `.content-card`。
- 关键样式集中在 `src/styles/global.css`：
  - `.content-card-grid` 当前为双列 `repeat(2, minmax(0, 1fr))`，`gap: 16px`。
  - `.content-card` 当前统一 `padding: 18px`。
  - `.quick-action-grid` 与 `.quick-action-grid__item` 控制“系统入口”内部项密度（当前内边距偏紧）。
  - 现有规则是全局通用规则，直接改会连带影响其他页面。

## Proposed Changes
- 文件：`src/pages/SettingsPage.tsx`
  - 在红框区域外层增加设置页专属容器类名（例如：`settings-page__focus-zone`），包裹：
    - 第一排 `.content-card-grid`
    - “环境信息”卡片
    - “第三方账号绑定”卡片
  - 目的：为本页“放大”需求提供局部作用域，避免污染全局。

- 文件：`src/styles/global.css`
  - 新增仅作用于 `SettingsPage` 的样式分组（以 `settings-page--revamp` + `settings-page__focus-zone` 为前缀）：
    - 放大整体体量：提高红框区卡片内边距（如 `18px -> 22/24px` 级别）。
    - 放大第一排布局：调整第一排网格间距（如 `16px -> 18/20px`），并确保左右列宽视觉更“铺满”。
    - 对齐上方窗格：为红框区首排卡设置最小高度，使其与顶部统计条形成统一节奏。
    - 放大“系统入口”内部项：增加 `.quick-action-grid__item` 内边距与行高，避免外层放大后内部显得偏小。
    - 放大列表信息密度：适度提高 `.settings-page__list > div` 的垂直 padding 与字体视觉层级，保持一致的“变大”感知。
  - 响应式保护：
    - 保持现有 `@media (max-width: 960px)` 单列逻辑不变。
    - 在移动端限制最小高度，防止首屏过长与信息折叠感过强。

## Assumptions & Decisions
- 已确认偏好：本次采用“高度 + 宽度都调”，且作用范围是“红框内全部卡片”。
- 决策：不直接修改全局 `.content-card` 基础规则，而使用设置页专属作用域覆盖，避免跨页面回归风险。
- 决策：以“小步增量”方式放大（优先 10%~25% 的尺寸提升），先达到与上方窗格对齐，再避免过度臃肿。

## Verification Steps
- 视觉验收（桌面端）：
  - 进入设置页，确认红框第一排两卡更高更宽，左右边界与上方统计卡观感对齐。
  - 确认“环境信息”“第三方账号绑定”卡片与第一排体量统一，不再显得偏瘦。
  - 确认“系统入口”内部 action 项尺寸同步放大，不出现“外大内小”。
- 响应式验收（<=960px）：
  - 卡片单列后无溢出、无异常空白、无文字截断。
- 回归检查：
  - 抽查 `HomePage`、`OperationsDashboardPage` 等使用全局 `content-card` 的页面，确认无样式连带变化。
