# 新增微信开放平台第三方登录实施计划

## Summary
- 目标：在现有账号体系中新增「微信开放平台网站应用扫码登录」，并覆盖登录弹窗、回调页、设置页绑定/解绑全链路。
- 范围：仅新增 `wechat` Provider，不替换现有 `onelinkai` 登录能力；保持用户名密码登录与现有数据结构兼容。
- 成功标准：
  - 登录页可发起微信扫码登录；
  - 回调成功后自动登录或自动建号；
  - 设置页可查看微信绑定状态并解绑；
  - 本地环境可跑通完整链路（含配置、接口、前端跳转）。

## Current State Analysis
- 后端已存在社交登录接口骨架：`/api/v1/auth/social/{provider}`、`/api/v1/auth/social/callback/{provider}`、`/api/v1/auth/social/links`、`/api/v1/auth/social/unbind`（`aigc-server/src/main/java/com/example/aigc/controller/AuthController.java`）。
- 当前实际逻辑硬编码为 `onelinkai`：`SocialAuthService` 中 `normalizeProvider` 与配置读取仅支持 `onelinkai`（`aigc-server/src/main/java/com/example/aigc/service/SocialAuthService.java`）。
- 配置层当前仅有 `aigc.auth.social.onelinkai`（`aigc-server/src/main/java/com/example/aigc/config/AuthProperties.java`、`aigc-server/src/main/resources/application.yml`）。
- 前端登录与回调页硬编码 `onelinkai`：
  - 登录按钮：`aigc-site-react/src/components/common/LoginModal.tsx`
  - 回调页：`aigc-site-react/src/pages/SocialLoginCallbackPage.tsx`
  - 路由：`aigc-site-react/src/router.tsx`
  - 设置页绑定管理：`aigc-site-react/src/pages/SettingsPage.tsx`
- 数据层已支持通用第三方绑定（`provider`、`provider_user_id`、`social_account` 表），无需新增 migration（`aigc-server/src/main/resources/db/migration/V26__add_onelink_social_login.sql`）。

## Assumptions & Decisions
- 已确认业务决策：
  - 使用「微信开放平台网站应用登录」（非企业微信）。
  - 首期范围为：登录弹窗 + 回调页 + 设置页绑定/解绑。
  - 账号策略：仅按微信唯一ID自动建号/登录，不做邮箱自动合并。
  - 环境策略：先打通本地开发联调，再扩展生产参数。
- 用户唯一标识策略：
  - 优先使用 `unionid`（若微信返回）；
  - 若未返回 `unionid`，回退 `openid`；
  - 存储在 `provider_user_id`，`provider` 固定为 `wechat`。
- 用户资料策略：
  - 用户名默认 `wechat_{providerUserId}`（带去重后缀）；
  - 显示名优先微信昵称，其次用户名；
  - 不引入额外必填资料页（保持最小实现）。

## Proposed Changes
### 1. 微信开放平台准备与配置（先做）
- 文件：`aigc-server/src/main/resources/application.yml`
- 文件：`aigc-server/src/main/java/com/example/aigc/config/AuthProperties.java`
- 变更内容：
  - 新增 `aigc.auth.social.wechat.*` 配置段，字段包含：
    - `enabled`
    - `app-id`
    - `app-secret`
    - `authorize-uri`（默认 `https://open.weixin.qq.com/connect/qrconnect`）
    - `token-uri`（默认 `https://api.weixin.qq.com/sns/oauth2/access_token`）
    - `user-info-uri`（默认 `https://api.weixin.qq.com/sns/userinfo`）
    - `redirect-uri`（本地默认 `http://localhost:5173/social/callback/wechat`）
    - `scope`（默认 `snsapi_login`）
  - 将配置绑定扩展到 `AuthProperties.SocialProperties`，新增 `WechatProperties`。
- Why：当前仅 `onelinkai`，无法注入微信 AppId/AppSecret 与微信官方端点。
- How：保持 `OnelinkaiProperties` 不动，仅新增并行 `WechatProperties`，减少回归风险。

### 2. 后端社交登录服务改造成多 Provider 可扩展
- 文件：`aigc-server/src/main/java/com/example/aigc/service/SocialAuthService.java`
- 文件：`aigc-server/src/main/java/com/example/aigc/service/social/SocialProvider.java`
- 新增文件：`aigc-server/src/main/java/com/example/aigc/service/social/WechatSocialProvider.java`
- 变更内容：
  - 引入 `SocialProvider` 注册表（`Map<String, SocialProvider>`），由 `provider` 动态路由。
  - `SocialAuthService` 去掉仅支持 `onelinkai` 的硬编码校验，改为：
    - 校验 provider 是否存在且启用；
    - 通过 provider 实现生成授权 URL、换 token、获取用户信息。
  - 新增 `WechatSocialProvider`，实现微信三步：
    - 授权地址拼接（`appid`, `redirect_uri`, `response_type=code`, `scope`, `state`, `#wechat_redirect`）；
    - 通过 `code` 获取 `access_token/openid/unionid`；
    - 通过 `access_token + openid` 拉用户资料，提取昵称、头像。
  - 用户标识映射：
    - `provider=wechat`
    - `provider_user_id=unionid(优先) | openid(回退)`
- Why：当前虽然有 `SocialProvider` 接口，但主流程未使用，必须先打通可扩展框架再加微信。
- How：以“最小重构”方式替换 provider 选择逻辑，保持现有 JWT 颁发与绑定关系复用。

### 3. 前端新增微信登录入口与通用回调处理
- 文件：`aigc-site-react/src/components/common/LoginModal.tsx`
- 文件：`aigc-site-react/src/pages/SocialLoginCallbackPage.tsx`
- 文件：`aigc-site-react/src/router.tsx`
- 文件：`aigc-site-react/src/api/index.ts`
- 变更内容：
  - 登录弹窗增加“使用微信登录”按钮，调用 `getSocialAuthUrl('wechat')`。
  - 回调页改为从路由参数识别 provider，不再硬编码 `onelinkai`。
  - 新增路由 `social/callback/wechat`（或改造成 `social/callback/:provider` 单路由）。
  - 回调成功后沿用现有 `socialLoginCallback(provider, code, state)` 完成 token 落地。
- Why：当前前端路径和文案均绑定 `onelinkai`，不能承载微信回调。
- How：优先采用“通用回调页 + provider 参数化”，避免后续每个 provider 复制一份页面。

### 4. 设置页新增微信绑定状态与解绑操作
- 文件：`aigc-site-react/src/pages/SettingsPage.tsx`
- 变更内容：
  - 在现有第三方绑定区块中新增微信项：
    - 未绑定：显示“绑定微信”按钮，触发授权跳转；
    - 已绑定：显示微信唯一ID（可部分脱敏）与“解绑微信”按钮。
  - 解绑复用 `/api/v1/auth/social/unbind`，参数传 `provider='wechat'`。
- Why：你要求首期包含绑定管理闭环。
- How：沿用现有 OneLinkAI 状态查询逻辑，抽成 provider 可复用渲染，避免重复代码。

### 5. 本地联调配置与操作说明（一步一步）
- 文件：`aigc-server/src/main/resources/application.yml`
- 文件：前端 `.env`（若项目已有示例文件，按现有方式补 `VITE_API_BASE_URL`）
- 操作步骤：
  - 第 1 步：在微信开放平台创建网站应用，配置回调域名（本地可用内网穿透域名，不可直接填 `localhost`）。
  - 第 2 步：获取 `AppID/AppSecret`，写入后端环境变量：
    - `AIGC_AUTH_SOCIAL_WECHAT_ENABLED=true`
    - `AIGC_AUTH_SOCIAL_WECHAT_APP_ID=...`
    - `AIGC_AUTH_SOCIAL_WECHAT_APP_SECRET=...`
    - `AIGC_AUTH_SOCIAL_WECHAT_REDIRECT_URI=http://localhost:5173/social/callback/wechat`（若用穿透域名，则填对应 HTTPS 回调）
  - 第 3 步：启动后端并确认 `/api/v1/auth/social/wechat` 返回授权地址。
  - 第 4 步：启动前端，打开登录弹窗，点击“使用微信登录”，手机扫码授权。
  - 第 5 步：回调页自动完成登录，回到首页后检查用户态。
  - 第 6 步：进入设置页，验证微信绑定状态与解绑流程。
- Why：你明确要求“教会一步一步操作与配置”。
- How：将平台申请、环境变量、接口联调、页面验证串成固定流水线，便于你复用到测试/生产。

## Verification Steps
- 配置校验：
  - `wechat.enabled=true` 且 `app-id/app-secret/redirect-uri` 非空时，授权接口返回可跳转 URL。
  - 缺少关键配置时，后端返回明确错误信息（不泄露密钥）。
- 登录校验：
  - 正常扫码授权后可成功登录并获取 JWT；
  - 重复扫码同一微信账号，不会重复创建本地用户。
- 绑定校验：
  - 已登录用户可绑定微信；
  - 解绑后在 `social/links` 中消失；
  - 若账号仅剩微信登录方式，解绑被安全规则拦截（与当前策略一致）。
- 回归校验：
  - 用户名密码登录不受影响；
  - OneLinkAI 登录入口与回调不受影响。
