# 组织用户页面收口与后台密度治理计划

## Summary
- 目标：修复“组织与用户”页面排版拥挤和功能堆叠问题，把低频/创建类功能改为按钮触发的抽屉或弹窗；同时对其它后台管理页做“内容爆表”检查并收口。
- 范围：按你确认的“仅后台管理页”执行，采用“单一入口优先”原则，做到功能不重不漏。

## Current State Analysis
- 组织与用户页当前问题（`aigc-site-react/src/pages/AdminDirectoryPage.tsx`）：
  - 页面同时展示：筛选区、批量操作、大表格、组织概览、详情区、快捷录入组织、快捷录入用户，首屏信息量过大。
  - 右侧栏堆叠了“详情区 + 两个快捷录入面板”，与左侧大表格并行，视觉重心分散。
  - 批量操作条直接常驻在表格上方，操作链路和阅读链路混在一起。
  - 详情区日志与基础信息同层展开，阅读成本高。
- 样式现状（`aigc-site-react/src/styles/components.css`）：
  - `admin-directory-page__layout` 为双栏 1.55fr + 0.9fr，右侧 sticky 区承载过多模块，导致“窄栏塞功能”。
  - 多个 `.teaching-panel` 连续堆叠，版块密度高，操作按钮分散在多个区域。
- 其它后台管理页审查结论：
  - `ProviderHubPage.tsx`：连接+模型双区块 + 多批量按钮，存在“动作入口分散”风险。
  - `ModelConfigPage.tsx`：`quick/advanced` 双模式并存，连接/模型入口重复。
  - `AuditLogsPage.tsx`、`MediaResourcesPage.tsx`：信息密度可接受，暂不作为重点改造对象。

## Proposed Changes

### 1) 组织与用户页：重排为“主工作台 + 右侧详情”，创建类全部弹窗化
- 文件：
  - `aigc-site-react/src/pages/AdminDirectoryPage.tsx`
  - `aigc-site-react/src/styles/components.css`
  - `aigc-site-react/src/components/common/ActionDrawer.tsx`（复用，不改行为）
- 变更内容：
  - 移除右侧常驻“快捷录入组织/班级”“快捷录入用户”面板。
  - 在主区域头部新增主按钮：`新建组织`、`新建用户`，分别打开 `ActionDrawer`。
  - 抽屉内复用现有表单字段和校验逻辑（不改 API，不改提交规则）。
- 目的：
  - 首屏只保留高频“筛选 + 表格 + 详情”，低频创建动作按需打开。

### 2) 批量操作改为按钮弹窗，避免常驻挤占表格
- 文件：
  - `aigc-site-react/src/pages/AdminDirectoryPage.tsx`
  - `aigc-site-react/src/components/common/ConfirmDialog.tsx`（复用）
- 变更内容：
  - 将现有批量启停条（`admin-directory-page__batch-bar`）改为“批量操作”按钮触发弹窗。
  - 弹窗中统一展示：已选数量、将启用/停用数量、确认按钮。
  - 表格区域仅保留轻量“已选 X 条”的状态标签，不再展示大块说明栏。
- 目的：
  - 将危险/批量动作收敛到确认弹层，减少视觉噪音与误触。

### 3) 详情区日志折叠为“查看日志”弹窗
- 文件：
  - `aigc-site-react/src/pages/AdminDirectoryPage.tsx`
- 变更内容：
  - 详情区默认仅显示“基础信息 + 校园身份”。
  - 操作日志从详情区移出，改为按钮打开弹窗（沿用当前 `detailLogs` 数据源）。
  - 保留“查看完整日志”跳转到审计页入口，确保功能不遗漏。
- 目的：
  - 详情区从“信息+日志混排”变成“信息优先，日志按需查看”。

### 4) 其它后台页面“内容爆表”治理（单一入口优先）
- 文件：
  - `aigc-site-react/src/pages/ProviderHubPage.tsx`
  - `aigc-site-react/src/pages/ModelConfigPage.tsx`
- 变更内容：
  - `ProviderHubPage`：将低频批处理动作（批量健康检查、批量导入）并入一个“更多操作”区（按钮/弹窗），避免与主操作并列堆叠。
  - `ModelConfigPage`：保持“快捷配置”为主入口；“高级配置”只保留一个显式入口，减少 quick/advanced 同层重复按钮。
  - 审计/资源页本轮只做检查不改结构（当前未明显爆表）。
- 目的：
  - 后台页面整体遵循同一密度规则：高频主动作常驻，低频治理动作进入二级入口。

## Assumptions & Decisions
- 决策 1：范围仅后台管理页，不扩散到教学/项目/创作工作台页面。
- 决策 2：组织与用户页默认隐藏并弹窗化的功能为：
  - 快捷录入组织/班级
  - 快捷录入用户
  - 批量操作条
  - 详情区日志
- 决策 3：采用“单一入口优先”，同一动作只保留一个主入口，其余改为跳转或次级入口。
- 决策 4：不改后端接口与数据结构，仅重排前端交互承载方式。

## Verification steps

    - 详情区默认不展开日志，点击按钮可查看日志弹窗；

- 其它后台页抽查：
  - `/models/hub`：主操作清晰，低频批处理不再并列堆叠。
  - `/models`：入口不重复，quick/advanced 层级清晰。
- 工具验证：
  - `cd aigc-site-react && npm run lint`
  - `cd aigc-site-react && npm run build`
