# 最终交付指南（Final Delivery Guide）

## 1. 任务启动步骤（Start-up）

1. 明确任务边界：复述需求目标、交付物、非目标范围，避免隐式扩展需求。
2. 建立上下文：先阅读实现细节索引，定位可能涉及的页面、Store、API、副作用入口与高风险区。
3. 执行改动前 Gate：按 Change Gate 模板完整填写 A-F 六个章节，并给出 `GO/NO-GO` 结论。
4. 生成执行清单：将计划改动文件、验证路径、回滚方式写成可执行 checklist 后再进入实现。

## 2. 改代码准入条件（Code Change Gate）

满足以下条件后，才允许进入代码改动：

- 已完成 Gate 模板 A-F 全章节，且结论为 `GO`。
- 已说明“拟改动点”与“不改动点”，避免范围漂移。
- 已覆盖上下游影响：至少说明直接影响、间接影响与回归风险点。
- 已定义最小验证路径：静态检查、自动化测试或关键手测路径可执行。
- 已定义可回滚策略：出现回归时可快速止损并恢复。

任一条件不满足时，状态应为 `NO-GO`，禁止直接改代码。

## 3. 完成后自证口径（Proof of Completion）

完成开发后，按以下口径提交自证：

1. 变更说明：列出实际改动文件、核心行为变化、未改动范围。
2. 证据清单：给出 lint/test/手测结果，包含执行命令与关键结果摘要。
3. 风险复盘：逐条对照 Gate 中的风险点，说明是否命中及处理结果。
4. 回滚可用性：确认回滚方案已验证或可直接执行。
5. 清单闭环：对照交付 checklist 勾选完成项，未完成项需说明原因与后续计划。
