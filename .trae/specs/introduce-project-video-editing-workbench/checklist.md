- [x] 项目首次进入剪辑工作台时，系统会基于可用镜头结果自动生成默认剪辑草稿
- [x] 默认草稿会优先选择口型同步结果，缺失时回退到镜头视频结果
- [x] 剪辑草稿可持久化保存片段顺序、启用状态、来源选择、入点、出点和简单转场
- [x] 前端能明确提示“草稿已修改但尚未发布”的状态
- [x] 用户可以在不立即渲染的情况下保存草稿，并在重新进入页面后恢复
- [x] 用户可以发起项目级剪辑预览渲染，并看到任务状态、失败原因和结果预览
- [x] 用户可以将某次可用预览结果发布为项目最终成片
- [x] 已发布成片会通过统一媒体资源抽象保存，并可在项目内直接预览或打开
- [x] 导出页与交付状态会优先消费最近一次已发布剪辑成片
- [x] 当不存在启用片段、片段来源失效或裁切区间非法时，后端会拒绝保存或渲染并返回明确错误
- [x] 无手动编辑时，系统仍可使用默认草稿快速生成成片，不阻断现有项目出片链路
- [x] 前后端最小验证覆盖草稿生成、草稿保存、非法输入、预览渲染、发布成片与导出页消费优先级
