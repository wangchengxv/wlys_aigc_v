# 高校 AIGC 实训平台 UI 精修 Spec

## Why
上一轮改版已经完成高校后台化的信息架构与功能布局，但整体视觉仍偏“可用优先”，页面质感、品牌识别度、组件一致性和重点信息表达还不够强。
本次希望继续推进下一步，将平台打磨成更好看、更有产品感的高校 AIGC 实训后台，提升观感、易读性和专业度。

## What Changes
- 为高校 AIGC 实训平台建立更统一、更有质感的视觉语言，包括色彩体系、卡片层级、阴影、圆角、间距、图标与状态标签规范
- 精修后台壳层与首页概览视觉，强化品牌区、导航选中态、指标卡片、快捷入口、待办与活动区的高级感
- 统一列表页、筛选区、表格区、详情抽屉、分页区与空状态的 UI 规范，使组织用户、课程实训、项目治理等页面风格一致
- 优化关键高频页面的视觉重点，包括平台概览、组织与用户、课程管理、作业提交、项目与作品、设置页
- 为平台补齐更适合高校与 AIGC 场景的视觉细节，例如校园风品牌色、AI 质感图形点缀、项目状态进度表现、作品封面信息卡
- 在不牺牲可读性与性能的前提下增加轻量级交互动效与反馈，例如 hover、切换、展开、抽屉过渡与骨架占位

## Impact
- Affected specs: `revamp-campus-aigc-admin-console`
- Affected code: `aigc-site-react/src/layouts/*`, `aigc-site-react/src/components/layout/*`, `aigc-site-react/src/components/common/*`, `aigc-site-react/src/components/script/*`, `aigc-site-react/src/pages/*`, `aigc-site-react/src/styles/*`

## ADDED Requirements
### Requirement: 平台具备统一的高质感后台视觉体系
系统 SHALL 提供一套适用于高校 AIGC 实训平台的统一视觉规范，使用户在不同后台页面之间切换时能感受到一致、专业、现代的产品观感。

#### Scenario: 浏览不同后台页面
- **WHEN** 用户在平台概览、组织与用户、课程实训、项目与作品、设置等页面之间切换
- **THEN** 页面使用统一的品牌色、背景层级、卡片样式、阴影、圆角、标题层级、状态标签和按钮语言
- **THEN** 不同页面的视觉密度、模块间距和重点信息表达保持一致

### Requirement: 首页概览页具备更强的视觉吸引力与信息优先级
系统 SHALL 将平台概览页打磨为更有产品感的后台首页，使关键指标、快捷入口、待办事项和最近活动具备更清晰的信息层级与视觉焦点。

#### Scenario: 打开平台概览页
- **WHEN** 管理员或教师进入平台概览
- **THEN** 首页优先突出核心指标卡片，并通过配色、图标、趋势或强调信息提升可扫描性
- **THEN** 快捷入口、待办事项、状态分布和最近活动区块具有明显而统一的视觉分层
- **THEN** 页面在桌面端下呈现更完整的后台首页质感，而不是简单卡片堆叠

### Requirement: 列表页采用统一的后台高可读布局
系统 SHALL 为列表型页面提供统一的高可读 UI，包括筛选栏、工具栏、数据表格、分页、空状态和批量操作反馈，以提升管理页面的稳定性和专业感。

#### Scenario: 浏览组织、课程或项目列表
- **WHEN** 用户打开组织与用户、课程、项目等列表页
- **THEN** 筛选区、操作区、表格区和分页区采用统一的布局结构和视觉样式
- **THEN** 表格的表头、行高、选中态、状态标签、操作按钮和 hover 反馈保持一致
- **THEN** 当页面无数据时展示设计完整的空状态，而不是简陋文本占位

### Requirement: 详情区与抽屉具备信息分层与展示质感
系统 SHALL 统一详情页和抽屉的版块结构、字段排版和视觉重点，使用户在查看用户详情、课程详情、项目详情时更容易理解关键信息。

#### Scenario: 查看详情信息
- **WHEN** 用户打开用户详情、课程详情或项目详情
- **THEN** 页面按版块展示基础信息、业务信息、状态信息、操作记录等内容
- **THEN** 重要字段具有更明显的视觉强调，非重点字段保持克制排版
- **THEN** 详情区域与列表页面在视觉上属于同一设计体系

### Requirement: 关键交互具备轻量动效与反馈
系统 SHALL 提供轻量、克制的交互动效和状态反馈，增强页面品质感，但不得影响高校后台使用效率和性能表现。

#### Scenario: 与后台界面交互
- **WHEN** 用户进行导航切换、卡片 hover、抽屉打开、分页切换或状态筛选
- **THEN** 系统提供一致且轻量的过渡、hover 反馈和加载占位
- **THEN** 动效时长和表现保持克制，不影响管理端高频操作

## MODIFIED Requirements
### Requirement: 后台页面从“可用”提升到“产品化”
现有后台页面 SHALL 在保持既有功能和信息架构不变的前提下，继续向更完整的产品化 UI 演进，重点优化视觉表现、页面节奏与组件一致性。

#### Scenario: 对比改版前后的后台页面
- **WHEN** 用户浏览改版后的高校 AIGC 实训平台
- **THEN** 页面相比当前版本具备更强的品牌感、秩序感和精致度
- **THEN** 用户能够更快识别重点信息、关键操作和状态变化

## REMOVED Requirements
### Requirement: 仅以功能齐全作为后台页面完成标准
**Reason**: 当前阶段平台已经具备较完整的功能布局，需要进一步通过优秀 UI 提升体验、专业度与展示效果。
**Migration**: 后续页面交付除满足功能完整外，还需满足视觉统一、重点突出、交互顺滑与页面观感达标的 UI 标准。
