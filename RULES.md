# RULES - 强制红线（法律）

## 1. 安全红线
- 禁止任何绕过登录态或项目归属校验的实现。
- 禁止将模型 API Key、JWT Secret 等敏感配置暴露给前端。
- 禁止在无校验情况下接收并落库用户上传文件。

## 2. 数据红线
- 删除默认逻辑删除；物理删除必须走异步清理策略。
- 涉及跨表写入必须保证事务一致性。
- 异步任务状态更新必须幂等，禁止重复写导致状态回退。

## 3. 架构红线
- 长耗时 AI/渲染任务必须异步化，不得阻塞主请求线程。
- AI 模型接入必须通过统一适配层，不得在业务层散乱硬编码第三方调用。
- Controller 禁止直接返回 Entity；必须经 DTO/Response 封装。

## 4. 业务红线
- 删除项目、删除资产、删除主体必须二次确认。
- 主体被分镜引用时，删除必须提示影响范围。
- AI 任务失败必须可见失败原因并支持重试。

## 5. 文档与测试红线
- 需求变更不更新 `SPECS` 视为不合规。
- 架构边界变更不更新 `ARCHITECTURE.md` 视为不合规。
- 未执行最小测试集或无结果记录，不得宣称“已完成”。
